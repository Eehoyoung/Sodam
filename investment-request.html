소담(SODAM)은 직원 1~5명 규모의 가맹점을 위한 Android+iOS 모바일 운영 비서입니다. NFC + GPS 출퇴근 인증, 급여·명세서 자동 계산, 노무·세무 자료 정리를 통해 본사는 운영 기준을 표준화하고 가맹점주는 월말 업무를 줄일 수 있습니다.

제안 요약

본 문서의 비용은 제안용 가정입니다. 실제 계약 시 가맹점 수, 본사 일괄 결제 여부, iOS 배포 범위, 커스터마이징 범위, 세무·노무 자문 포함 여부에 따라 조정됩니다.

가맹점 운영에서 반복되는 문제

가맹점마다 출퇴근 확인 방식과 급여 계산 방식이 다르면 본사는 현장 리스크를 늦게 발견합니다.

01

출퇴근 증빙 불일치

어떤 매장은 카톡, 어떤 매장은 수기 노트, 어떤 매장은 사진 인증을 사용해 본사가 통합 확인하기 어렵습니다.

02

급여 계산 기준 차이

주휴수당, 야간·연장수당, 퇴직금 계산 기준이 가맹점마다 달라 분쟁과 민원 리스크가 생깁니다.

03

기기 환경 차이

점주와 직원이 Android와 iPhone을 섞어 쓰기 때문에 한쪽 OS만 지원하면 현장 도입률이 떨어질 수 있습니다.

소담의 해결 방식

가맹점은 Android/iOS 앱으로 쉽게 쓰고, 본사는 표준화된 데이터를 기준으로 운영 상태를 확인합니다.

NFC + GPS 이중 출퇴근 인증

매장 NFC 태그와 GPS 반경 검증을 결합해 현장 출근 여부를 확인합니다. 대리 출근과 위치 조작 리스크를 줄입니다.

급여·명세서 자동 계산

기본 시급, 직원별 시급, 연장·야간·주휴수당 계산을 연결하고 사장/직원 모두 급여 명세를 확인할 수 있게 합니다.

본사 표준 운영 리포트

가맹점별 근무시간, 예상 인건비, 명세서 발급 여부, 누락 출퇴근을 본사 기준으로 집계할 수 있습니다.

Android+iOS 동시 도입

점주·직원 기기 환경에 관계없이 사용할 수 있도록 양대 모바일 OS를 지원하는 방향으로 출시와 QA를 설계합니다.

가맹점별 예상 지출 비용

가맹점 1곳 기준 예상 비용입니다. Android/iOS 앱 사용료는 동일하게 책정하고, iOS 지원으로 인한 추가 비용은 본사 계약·초기 구축 범위에서 별도 협의할 수 있습니다.

프랜차이즈 본사 도입 가치

소담은 단순한 가맹점 편의 앱이 아니라, 본사가 가맹점 운영 리스크를 표준화해서 관리할 수 있는 현장 데이터 인프라입니다.

운영 표준화

출퇴근, 급여, 명세서 발급 기준을 본사 가이드에 맞춰 통일합니다.

리스크 조기 발견

누락 출퇴근, 과도한 인건비, 급여명세서 미발급 등 문제 징후를 조기에 확인합니다.

기기 제약 완화

Android와 iPhone 사용자를 모두 고려해 가맹점 도입 장벽을 낮춥니다.

도입 구조 예시

프랜차이즈 본사와의 계약은 가맹점 직접 결제형, 본사 일괄 결제형, 혼합형으로 설계할 수 있습니다.

도입 방식비용 부담장점추천 상황
가맹점 직접 결제형각 가맹점본사 초기 부담이 낮고 자율 도입 가능파일럿 테스트, 일부 매장 우선 도입
본사 일괄 결제형본사전체 가맹점 표준화가 빠르고 데이터 품질이 균일본사가 운영 기준을 강하게 통일하려는 경우
혼합형본사 기본 지원 + 가맹점 유료 업그레이드기본 기능은 통일하고 고급 기능은 선택 과금 가능가맹점 규모와 기기 환경 편차가 큰 브랜드

현재 진행 상황

레포 기준으로 백엔드·프론트·문서 체계가 구성되어 있으며, Android와 iOS 베타 출시를 목표로 외부 Live 키와 운영 검증 단계가 남아 있습니다.

BE

Spring Boot API

Java 17, Spring Boot, JPA, MySQL/Redis 기반. OpenAPI 명세와 인증·근태·급여·구독 도메인이 구성되어 있습니다.

APP

React Native 앱

Android와 iOS 양쪽 배포를 고려한 모바일 앱. 사장/직원 화면, 출퇴근, 급여, 매장, 구독 화면을 중심으로 구성되어 있습니다.

OPS

운영 준비

Docker 풀스택, 개발 seed 계정, Swagger, Adminer, 문서 카탈로그를 통해 개발·운영 흐름을 정리했습니다.

프랜차이즈 도입 로드맵

한 번에 전 가맹점에 적용하기보다, Android+iOS 파일럿 매장에서 검증 후 확대하는 방식을 권장합니다.

1단계

파일럿 3~5개 매장

Android와 iOS 사용자를 모두 포함해 NFC/GPS 출퇴근, 급여 계산, 명세서 조회, 점주 사용성을 실제 현장에서 검증합니다.

2단계

본사 리포트 기준 확정

본사가 보고 싶은 가맹점별 인건비, 출퇴근 누락, 근무시간, 명세서 발급 지표를 확정합니다.

3단계

전체 가맹점 확대

도입 매뉴얼, 점주 교육자료, 직원 안내문을 표준화해 전체 가맹점으로 확산합니다.

주요 리스크와 대응

프랜차이즈 본사 제출용 문서인 만큼 도입 과정에서 예상되는 질문을 먼저 정리했습니다.

가맹점 사용 저항무료 또는 저가 플랜으로 시작하고, 월말 급여 정산 시간 절감 효과를 체감하게 합니다.
개인정보·위치정보 리스크근태 목적의 최소 데이터만 수집하고, 약관·개인정보·위치정보 동의 절차를 분리합니다.
iOS 심사·배포 리스크Android 선검증 후 iOS TestFlight와 App Store 심사 대응을 병행하고, 파일럿 매장에 iPhone 사용자를 반드시 포함합니다.
매장별 운영 편차가맹점 규모별 플랜을 나누고, 본사 일괄 도입 또는 선택 도입을 병행할 수 있게 설계합니다.
Closing Message

소담은 가맹점주의 업무 시간을 줄이고, 본사의 운영 기준을 선명하게 만듭니다.

출퇴근 기록 하나가 급여, 명세서, 노무 자료, 본사 리포트까지 이어지면 프랜차이즈 운영은 더 투명해집니다.
