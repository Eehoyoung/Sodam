본 문서의 시장 규모·매출 전망·일정은 내부 문서 기준의 목표 및 가정입니다. 실제 투자 검토 시 외부 시장 데이터, 회계 자료, 사용자 인터뷰, 테스트 결과를 별도로 검증해야 합니다.

문제: 작은 가게일수록 운영은 더 수작업입니다

사장님은 매일 영업을 하면서 출퇴근 확인, 급여 계산, 세무 준비까지 직접 처리해야 합니다.

01

출퇴근을 믿기 어렵다

카톡 보고, 사진 인증, 수기 노트는 대리 출근·부정 인증·누락 기록에 취약합니다.

02

급여 계산이 복잡하다

시급, 연장, 야간, 주휴수당, 퇴직금까지 매월 엑셀과 계산기에 의존합니다.

03

세무·노무 정보가 흩어져 있다

고용노동부, 국세청, 블로그, 세무사 상담 사이에서 사장님은 매번 다시 검색합니다.

솔루션: 사장님의 주머니 속 운영팀

소담은 근태 데이터에서 시작해 급여·명세서·세무 준비로 이어지는 흐름을 자동화합니다.

NFC + GPS 이중 출퇴근 인증

매장 NFC 태그와 GPS 반경 검증을 결합해 현장 출근 여부를 확인합니다. 부정 출퇴근 방지와 분쟁 증거력을 동시에 노립니다.

급여·명세서 자동 계산

기본 시급, 직원별 시급, 연장·야간·주휴수당 계산을 연결하고 사장/직원 모두 급여 명세를 확인할 수 있게 합니다.

노무·세무 인포허브

노무, 세무, 정책, 팁 콘텐츠를 앱 안에 모아 사장님이 필요한 순간 바로 참고할 수 있게 합니다.

세무 파트너 연결 모델

세무 대행을 직접 수행하지 않고, 자료 정리와 세무사 연결 중심의 정액 패키지 모델로 법적 리스크를 낮춥니다.

현재 진행 상황

레포 기준으로 백엔드·프론트·문서 체계가 구성되어 있으며, Android MVP 출시를 목표로 외부 Live 키와 운영 검증 단계가 남아 있습니다.

BE

Spring Boot API

Java 17, Spring Boot, JPA, MySQL/Redis 기반. OpenAPI 명세와 인증·근태·급여·구독 도메인이 구성되어 있습니다.

FE

React Native 앱

Android 우선 모바일 앱. 사장/직원 화면, 출퇴근, 급여, 매장, 구독 화면을 중심으로 구성되어 있습니다.

OPS

운영 준비

Docker 풀스택, 개발 seed 계정, Swagger, Adminer, 문서 카탈로그를 통해 개발·운영 흐름을 정리했습니다.

Investment Ask

요청 투자금: ₩25,000,000

예산 기준은 “현실적인 베타/투자자용 출시형”입니다. 창업자가 직접 개발을 주도하고, 법무·세무·디자인·QA·고객 인터뷰 영역만 선택적으로 외부 자원을 사용합니다.

M0 ~ M3

Android MVP 출시

무료/유료 플랜, NFC+GPS 출퇴근, 급여·명세서, 매장·직원 관리, 기본 구독 게이팅을 정식화합니다.

M4 ~ M8

수익화 및 세무 연결

세무사 파트너 1~2곳 확보, 세무 자료 정리 단건결제, 4대보험 서식, 운영 대시보드를 강화합니다.

M9 ~ M12

확장과 반복 성장

iOS·다매장·프리미엄 플랜·웹 대시보드 베타를 통해 객단가와 유지율을 끌어올립니다.

주요 리스크와 대응

투자자 관점에서 먼저 물어볼 질문을 숨기지 않고 정리했습니다.

1인 창업 리스크범위를 MVP로 제한하고, 결제·법무·세무·디자인 등 핵심 외부 영역은 필요한 시점에 외주/파트너로 보완합니다.
경쟁사 진입 리스크대형 HR SaaS가 덜 집중하는 직원 1~5명 초소형 사업장, 세무 준비, 현장형 모바일 UX로 포지셔닝합니다.
법무·개인정보 리스크주민번호·계좌번호 직접 저장을 지양하고, 세무 대행이 아닌 세무사 연결·자료 정리 모델로 설계합니다.
유료 전환 리스크명세서 PDF, 노동법 경고, 퇴직금, 운영 대시보드 등 사장님이 월말에 바로 돈을 낼 이유가 있는 기능을 유료 벽으로 배치합니다.
Closing Message

소담은 작은 가게 사장님의 시간을 사는 제품입니다.

출퇴근 기록 하나가 급여, 명세서, 세무 준비, 분쟁 증거까지 이어지는 구조를 만들면 작은 매장도 인사팀과 회계팀을 가진 것처럼 운영할 수 있습니다.
