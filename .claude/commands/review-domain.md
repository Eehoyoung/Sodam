# /review-domain — 도메인 정합성 검토

소담의 핵심 도메인 코드를 검토한다. 한국 노동법·세무 룰과 일치하는지 확인.

## 검토 대상

**기본**: 인자로 받은 도메인 (`payroll`, `attendance`, `wage`, `store`)
**기본값**: `payroll` (가장 매출 영향 큰 영역)

## 점검 항목

### Payroll (급여)
- 최저시급 갱신 반영 (2026년 ₩10,030 가정)
- 주휴수당: 주 15시간 이상 + 결근 없을 시
- 야간(22:00-06:00): 1.5배 가중치
- 연장(주 40시간 초과): 1.5배 가중치
- 야간+연장 중복: 2.0배
- 사업소득세 원천징수 3.3%

### Attendance (출퇴근)
- NFC 태그 ID 검증
- GPS Haversine 반경 검증
- 운영시간 외 출근 경고 (차단 X)
- 수동 등록 시 사유 필수

### Wage (시급)
- 매장 기본 시급 > 0
- 직원별 시급 우선, 없으면 매장 기본
- 시급 변경 시 이력 보존

### Store (매장)
- 사업자번호 형식 검증
- 운영시간 OpenAt/OpenNow 로직
- Soft Delete 일관성

## 산출

1. 통과/실패 항목 표
2. 실패 케이스의 구체 코드 위치 (file:line)
3. 수정 패치 제안

## 호출 예
```
/review-domain payroll
/review-domain attendance
```
