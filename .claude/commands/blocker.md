# /blocker — 출시 차단 점검

## 작업 순서

1. **`FEATURES.md`** 의 "출시 차단 (Blocker) 체크리스트" 섹션을 읽는다.
2. **`마스터프로젝트개발계획.md`** §7 (출시 정의) 를 함께 점검한다.
3. 미완료 항목 중 가장 영향 큰 1개를 선택하고 사용자에게 보고한다.
4. 사용자 승인 후 해당 항목 진행:
   - 코드 작업이면 → 구현 + 단위 테스트
   - 외부 서비스 연동이면 → 코드 스켈레톤 + API 키 발급 안내
   - 정책/약관이면 → 초안 작성 (변호사 검토 필요 표시)
5. 진행 후 체크리스트에 ✅ 표시하고 다음 후보를 제시.
