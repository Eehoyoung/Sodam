## 0. 분석 방법

82개 목업과 `07-interaction-storyboards.md`(상호작용 전수), `09-copy-and-state-library.md`(빈/오류/성공 카피)를 교차 검토하고, 실제 코드에서 **로직은 이미 존재하나 대응하는 화면/상태가 목업에 없는 곳**을 식별했다. 근거가 코드/스펙에 없는 항목은 제외했다.

확인된 사실:
- `api.ts`에 401 → refresh 단일비행 큐가 있고, refresh 실패 시 `setOnUnauthorized`로 `AuthContext.refetchAuth()`만 호출 → **세션 만료 시 전용 안내/재로그인 화면 없이 조용히 인증 상태만 리셋**된다.
- `offlineAttendanceQueue.ts` + `useOfflineSync.ts`로 오프라인 출퇴근 큐/동기화 로직은 있으나, **목업 48 ErrorState(network) 외에 "오프라인 배너 / 동기화 대기 / 동기화 완료" 상태 화면이 정의되어 있지 않다.**
- `usePushPermission.ts`는 **stub(항상 granted)** 이며 권한 프라이머(왜 알림이 필요한지) 화면이 목업에 없다. `07` G-MS-003은 위치/카메라/NFC 프라이머만 다룬다.
- `SubscribeScreen`은 `TossBillingAuth`, `TaxRefundIntake` 라우트로 분기하지만, **결제 실패/재시도, 빌링키 등록 실패, 구독 만료·결제 누락, 환급형 신청 진행 상태** 화면이 목업·스펙에 없다(목업 31/71/72는 정상 흐름과 시트만 커버).
- 권한 **영구 거부(Don't ask again) 후 OS 설정으로 보내는 분기**가 49 PermissionState에 한 줄(`권한 켜기`)로만 있고 별도 카피가 없다.

## 2. 항목별 상세 + 카피 제안

### A1. 세션 만료 안내 + 재로그인 (P0)
`api.ts`의 refresh 실패는 토큰을 지우고 인증 상태만 되돌린다. 사용자는 작업 중 갑자기 로그인 화면으로 떨어진다. 작업 중이던 데이터 손실 우려와 재진입 경로를 명확히 줘야 한다.
> 제목: **다시 로그인해 주세요**
> 설명: 보안을 위해 일정 시간이 지나면 자동으로 로그아웃돼요. 작성 중이던 내용은 저장돼 있으니 안심하세요.
> CTA: `다시 로그인` / 보조: `고객지원 보기`

### A2. 글로벌 오프라인 배너 + 동기화 상태 (P0)
오프라인 출퇴근 큐가 동작하지만 사용자에게 "지금 오프라인 → 대기 중 → 연결되어 처리됨"이 보이지 않으면 "출근이 됐나?" 불안이 남는다. 상단 슬림 배너 + 토스트로 처리.
> 오프라인 배너: **지금 오프라인이에요. 기록은 안전하게 보관 중이에요.**
> 대기 토스트: **연결되면 출근 기록을 자동으로 올릴게요.**
> 복구 토스트: **다시 연결됐어요. 대기 중이던 출근 1건을 처리했어요.**

### A3. 권한 영구거부(설정 이동) 분기 (P0)
GPS/NFC/카메라를 "다시 묻지 않음"으로 거부하면 앱 내 권한요청이 막힌다. 49 PermissionState를 두 가지(요청 가능 / OS설정 필요)로 분리.
> 제목: **위치 권한이 꺼져 있어요**
> 설명: 출근 위치를 확인하려면 휴대폰 설정에서 위치 권한을 켜주세요. 켜기 어려우시면 사장님께 수동 출근을 요청할 수 있어요.
> CTA: `설정 열기` / 보조: `사장님께 수동 요청`

### A4. 결제 실패 / 재시도 결과 (P0)
`SubscribeScreen` → `TossBillingAuth`에서 빌링키 등록 또는 정기결제가 실패할 수 있다. 결제는 승인 필수 영역이므로 **금액/로직은 건드리지 않되 결과 표현 화면**만 추가.
> 제목: **결제를 마치지 못했어요**
> 설명: 카드 정보나 한도를 확인한 뒤 다시 시도해 주세요. 요금이 중복 청구되지는 않아요.
> CTA: `다시 결제하기` / 보조: `다른 결제 수단` / 3차: `고객지원 보기`

### A5. 구독 만료 / 결제 누락 차단 안내 (P0)
비즈니스 전용 기능(명세서 발급 등) 진입 시 구독이 없거나 만료면 막히는데, "왜 막혔고 어떻게 푸는지"가 없으면 이탈. gating 시 노출.
> 제목: **비즈니스 플랜에서 쓸 수 있어요**
> 설명: 명세서 발급은 비즈니스 플랜 기능이에요. 지금 시작하면 바로 직원에게 명세서를 보낼 수 있어요.
> CTA: `플랜 보기` / 보조: `나중에`
> (만료 케이스) 제목: **구독이 만료됐어요** / 설명: 결제 수단을 확인하면 기능을 다시 쓸 수 있어요. / CTA: `결제 수단 관리`

### A6. 콜드스타트 — 매장 0개 사장 첫 진입 (P1)
가입 직후 OwnerHome가 지표 0으로 차 있으면 막막하다. 홈 자체에 "첫 매장 등록" 단일 행동을 크게.
> 제목: **첫 매장을 등록해 볼까요?**
> 설명: 매장을 등록하면 직원 초대와 출퇴근, 급여 정산을 바로 시작할 수 있어요.
> CTA: `매장 등록하기`

### A7. 콜드스타트 — 직원 0명 매장 (P1)

> 설명: 초대 코드를 보내면 직원이 직접 가입하고 출근을 시작해요.
> CTA: `초대 코드 공유` / 보조: `초대 코드 복사`

### A8. 푸시 알림 수신 동의 프라이머 (P1)
