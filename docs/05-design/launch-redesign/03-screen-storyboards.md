# Screen Storyboards

## S-000 RoleStart

**목표**  
사용자가 자기 역할을 3초 안에 고르고, 가입/로그인을 시작한다.

**구성**
- 브랜드 심볼과 한 줄 가치 제안
- 역할 카드 3개: 사장님, 직원, 개인 기록
- 선택된 카드에는 실제 얻는 결과를 한 문장으로 표시
- 기본 CTA: `이 역할로 시작하기`
- 보조 CTA: `이미 계정이 있어요`

**상태**
- 미선택: CTA 비활성 대신 첫 번째 역할을 기본 추천
- 선택: 카드 확장, 하단 CTA 문구 변경
- 네트워크 없음: 로그인 전이므로 화면 자체는 동작, 인증 단계에서 안내

## O-001 OwnerHome

**목표**  
사장이 오늘 매장 상태를 1초 안에 파악하고, 문제를 바로 처리한다.

**구성**
- AppHeader: 매장명 드롭다운, 알림 아이콘
- Hero Summary: 오늘 해야 할 일 수, 현재 출근 인원, 이번 달 예상 인건비
- Priority Strip: 미출근, 누락 기록, 정산 준비 같은 예외
- Today Attendance: 직원별 상태 리스트
- Payroll Pulse: 이번 달 급여 예상/정산 준비율
- Quick Actions: 급여 정산, 직원 초대, 매장 설정, 노무 정보

**주요 CTA**
- 예외가 있으면 `이상 출퇴근 확인`
- 예외가 없으면 `급여 정산 시작`

**빈 상태**
- 매장 없음: `첫 매장 등록하기`
- 직원 없음: `직원 초대 코드 만들기`

## O-101 StoreDetail as Ops Console

**목표**  
매장 상세를 읽기 화면이 아니라 운영 콘솔로 만든다.

**구성**
- 매장 헤더: 이름, 주소, 인증 반경 상태
- 초대 코드 카드: 복사/공유
- 오늘 출근 요약
- 직원 리스트: 출근 상태, 시급, 이번 달 근무시간
- 설정 바로가기: 매장 정보, 위치 인증, 시급 정책, 운영 시간
- 하단 CTA: `직원 추가`

**상태**
- 위치 인증 미설정: 경고 카드와 `반경 설정`
- 직원 없음: 초대 코드 우선 노출

## O-301 PayrollRun

**목표**  
사장이 월말 급여 정산을 믿고 끝낸다.

**단계**
1. 매장/기간 선택
2. 미완료 출퇴근 확인
3. 직원별 자동 계산 미리보기
4. 가감 조정과 사유 입력
5. 총액 확인
6. 명세서 발급과 직원 알림

**핵심 UX**
- `매장 ID 입력`은 금지, 매장 선택 컴포넌트를 사용한다.
- 미완료 출퇴근이 있으면 정산 전 정리할 수 있게 연결한다.
- 계산 근거를 기본/연장/야간/주휴/세금/가감으로 분리한다.

## E-001 EmployeeHome

**목표**  
직원이 출근/퇴근을 5초 안에 끝낸다.

**구성**
- 인사/날짜
- 현재 매장과 시급
- 중앙 큰 CTA: 출근하기 / 근무 중 타이머 / 퇴근하기 / 오늘 근무 완료
- 인증 상태: GPS, NFC, 오프라인 큐
- Quick Links: 급여명세, 근무 캘린더, 매장 코드 입력

**상태**
- 매장 없음: 중앙 CTA를 `매장 코드 입력`으로 전환
- 위치 권한 없음: `위치 권한 켜기`와 수동 요청 옵션
- 반경 밖: 사유 안내와 정정 요청 진입

## E-101 AttendanceCalendar

**목표**  
직원이 내 기록이 정상인지 확인하고 이상 기록을 정정 요청한다.

**구성**
- 월 이동
- 월간 요약: 근무일, 총 시간, 예상 급여
- 캘린더: 정상/누락/휴가/정정 대기 점 표시
- 선택일 상세
- CTA: `정정 요청`

## E-301 JoinStoreByCode

**목표**  
신규 직원이 매장 코드로 빠르게 합류한다.

**구성**
- 코드 입력 6~8자리
- QR 스캔 영역
- 도움말: 사장님 앱의 초대 코드 위치
- 성공 화면: 매장명, 적용 시급, 오늘 출근 CTA

## P-001 PersonalHome

**목표**  
개인 사용자가 회사 승인 없이 내 근무 시간을 기록한다.

**구성**
- 개인 근무지 선택
- 오늘 기록 타이머
- 출근/휴게/퇴근 세그먼트 버튼
- 오늘 예상 급여
- 월간 요약
- 수동 기록 추가

**차별점**
- 매장 가입, 사장 승인, 정정 요청이 없다.
- 직접 수정과 개인 메모가 중심이다.

## M-001 MyPage

**목표**  
홈/대시보드가 아니라 계정, 구독, 알림, 지원의 허브로 정리한다.

**구성**
- 프로필
- 구독/결제 상태
- 알림 설정
- 계정 설정
- 문의/Q&A
- 약관/개인정보
- 로그아웃/탈퇴
