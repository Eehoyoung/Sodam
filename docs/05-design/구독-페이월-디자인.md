# 구독·페이월 디자인 — 4티어 수익화 v3.1

> 대상 컴포넌트: `frontend/src/features/subscription/components/`
> `SubscriptionPlanCard` · `BillingCycleToggle` · `PaywallSheet`
> 원칙: 모두 프레젠테이셔널(순수) — props 로만 동작, API·네비게이션·데이터 패칭 없음.

## 1. 4티어 위계

| 티어 | 이모지 | 가격(월) | 정원 | accent 토큰 | 비고 |
|---|---|---|---|---|---|
| FREE | 🌱 | 무료 | 직원 2 | `textSecondary` | 진입 |
| STARTER | ✨ | 9,900원 | 직원 5 | `brandPrimary` | 첫 유료 |
| PRO | 👑 | 19,900원 | 무제한 | `brandSecondary`(네이비) | **추천/앵커** |
| PREMIUM | 💎 | 39,900원 | 무제한 | `success` | 상위 |

- accent 색은 항상 `useThemeColors()` 에서 해석 → 다크 모드 안전.
- 가격은 컴포넌트가 포맷하지 않는다. `priceLabel`(예: `"무료"`, `"월 9,900원"`)로 **이미 포맷된 문자열**을 받는다.

### 끝자리-9 가격 사다리 (앵커링)
9,900 → 19,900 → 39,900 의 끝자리-9 계단으로 **PRO(19,900)** 가 STARTER 대비 "2배지만 무제한", PREMIUM 대비 "절반 가격" 으로 보이게 만든다. PRO 가 합리적 중간값으로 읽히도록 시각 강조를 PRO 에 집중한다.

## 2. SubscriptionPlanCard — 카드 레이아웃·위계

```
┌─────────────────────────────────────┐
│ 👑  프로                  [추천][이용 중] │   ← header: 이모지+이름+가격 / 우측 배지
│     월 19,900원                        │
│ ───────────────────────────────────── │   ← divider
│ ✓ 직원 무제한 + 풀 노동법 경고          │   ← included row
│ ✓ 4대보험 신고서 자동작성               │
│ – 멀티매장 무제한                       │   ← locked row (muted)
└─────────────────────────────────────┘
```

- **앵커(PRO) 강조**: `recommended` 카드는 `AppCard variant="elevated"`(그림자) + 선택 전에도 옅은 브랜드 ring(`borderColor: brandPrimaryMuted`, 1.5px). 선택 시 `AppCard` 의 굵은 브랜드 ring(2px) 으로 승격.
- 비앵커 카드는 `variant="flat"`.
- **이름 색**은 티어 accent, **가격**은 `caption/secondary` 로 보조 위계.
- 배지: 추천(`warning` 톤 — amber, 고대비 fg), 이용 중(`success` 톤). 우측 상단 세로 정렬.
- 카드 onPress → `selected` 토글은 호출 측 상태. 카드 자체는 `accessibilityRole="button"` + selected state (AppCard 기본).

### included / locked row 패턴
- **included**: `✓`(success color) + 본문 `primary` 톤.
- **locked**: `–`(textTertiary, muted) + 본문 `tertiary` 톤. ✕·취소선은 쓰지 않는다(부정 강조 대신 "아직 잠김"의 중립 톤).
- 아이콘 고정폭 22px 로 텍스트 좌측 정렬 일관.

## 3. BillingCycleToggle — 주기 토글·할인 표기

세그먼트 컨트롤: **월납 / 반년납 / 연납**.

| 값 | 라벨 | 할인 표기 |
|---|---|---|
| `MONTHLY` | 월납 | — |
| `HALF_YEARLY` | 반년납 | 1개월 무료 |
| `YEARLY` | 연납 | 2개월 무료 |

- 할인 문구는 비월납 세그먼트 **하단 보조 라인**(10px)으로 표기 → 연납으로 갈수록 이득이 커 보이는 affordance.
- 선택 세그먼트: 흰 배경(`background`) + `sm` 그림자 + 브랜드 accent 텍스트. 비선택: `surfaceMuted` 트랙 위 보조 텍스트.
- 할인 표기 2줄을 담기 위해 세그먼트 `minHeight: 48`(공통 SegmentedControl 36 보다 큼).
- 접근성: `accessibilityRole="tab"`, `accessibilityState.selected`, 라벨에 할인 포함("반년납, 1개월 무료").

## 4. PaywallSheet — 카피·상태

공통 `BottomSheet` 프리미티브 재사용(handle → title → description → content → primary → secondary).

- **title**: "잠깐, 이 기능은 잠겨 있어요"
- **description**: `이 기능은 {requiredPlanLabel} 플랜에서 이용할 수 있어요.` (사장님 존댓말)
- **content**: 🔒 + 서버 제공 사유(`message`) 를 warm 박스(`surfaceWarm` + `brandPrimaryMuted` border)로 표시.
- **hint**: "업그레이드하면 바로 이용할 수 있어요. 언제든 변경·해지할 수 있어요."
- **primary CTA**: `{requiredPlanLabel} 플랜으로 업그레이드` → `onUpgrade`.
- **secondary**: "닫기"(ghost) → `onClose`.

상태: `visible` 으로 표시/숨김. 백드롭/제스처 닫기는 `onClose` 로 위임(BottomSheet 기본).

## 5. 다크 모드 · compact 대응

- 모든 색은 `useThemeColors()` 에서 해석 → 라이트/다크 자동.
- **compact(<360)**: 카드 이모지 28→24, included row gap 한 단계 축소(카드 4장 세로 흐름 완화).
- 텍스트는 `useResponsive().pick()` 의 이산 값만 사용(폭 기반 폰트 스케일 금지 — OS 텍스트 배율 존중).
- 세그먼트 라벨/할인은 `numberOfLines={1}` 로 좁은 폭에서도 줄바꿈 깨짐 방지.

## 6. prop 계약 (FE 에이전트 import 기준)

```ts
// SubscriptionPlanCard
type PlanHighlight = {text: string; included: boolean};
type PlanCardView = {
  name: 'FREE'|'STARTER'|'PRO'|'PREMIUM';
  displayName: string; priceLabel: string; emoji: string;
  recommended?: boolean; highlights: PlanHighlight[];
};
interface SubscriptionPlanCardProps {
  view: PlanCardView; selected: boolean; isCurrent: boolean; onPress: () => void;
}

// BillingCycleToggle
type BillingCycleValue = 'MONTHLY'|'HALF_YEARLY'|'YEARLY';
interface BillingCycleToggleProps {
  value: BillingCycleValue; onChange: (v: BillingCycleValue) => void;
}

// PaywallSheet
interface PaywallSheetProps {
  visible: boolean; requiredPlanLabel: string; message: string;
  onUpgrade: () => void; onClose: () => void;
}
```
