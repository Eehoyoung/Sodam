# Settings · 알림 설정

## 목적
방해금지 시간대 + 알림 종류별 토글로 사용자가 푸시 받을 시점·종류 제어.

## 레이아웃
```
┌─────────────────────────────┐
│ ←  알림 설정                   │
├─────────────────────────────┤
│ 푸시 알림                     │  (sm secondary)
│                             │
│ ☑ 알림 받기                  │  ← 마스터 토글
│                             │
│ ─ 종류별 ─                   │
│ ☑ 출근/퇴근 알림              │
│ ☑ 급여 지급 알림              │
│ ☑ 결제 알림                  │
│ ☐ 프로모션·마케팅             │
│                             │
│ ─ 방해금지 시간 ─             │
│                             │
│ ☑ 방해금지 사용                │
│ 시작: [22:00]                │
│ 종료: [07:00]                │
│ (긴급 알림 제외)               │
│                             │
│ ─ 이메일 알림 (P2) ─          │
│ 향후 지원 예정                 │
└─────────────────────────────┘
```

## 토큰
- 토글: 28×16, brandPrimary 활성
- 시간 선택: TimePicker 모달 (DateTimePicker)
- 섹션 헤더: `textSecondary` 라벨

## 저장
- `AsyncStorage.notificationPrefs` JSON
- 변경 즉시 저장 (debounce 300ms)
- BE 동기화는 P1

## 엣지케이스
- 마스터 OFF → 종류별 모두 비활성 (회색 처리)
- 방해금지 종료=시작 → 24시간 차단으로 해석 (확인 모달)
