# S-201 · 직원 상세 (사장 뷰)

## 목적
한 직원의 모든 정보·이력을 4 탭으로 통합 표시. 시급 변경·비활성화 등 액션 진입.

## 레이아웃
```
┌─────────────────────────────┐
│ ←  지훈(데모) ⋮                │  ← Header + 오버플로우 메뉴
├─────────────────────────────┤
│  ┌──┐                       │
│  │지│  지훈(데모)              │
│  └──┘  staff@sodam.dev      │
│                             │
│  💰 시급 12,000원/h          │
│  📅 입사일 2025-12-01         │
│  🟢 활성                     │
├─────────────────────────────┤
│ [정보] 출퇴근  급여  연차       │  ← Tab bar (active=brandPrimary)
├─────────────────────────────┤
│  탭별 콘텐츠                  │
├─────────────────────────────┤
│ ─ 사용자 메모(사장만 보임) ─   │
│ "성실하고 마감 잘 함"          │
│                             │
│ [시급 변경] [비활성화]          │  ← 하단 Action Row
└─────────────────────────────┘
```

## 4탭 콘텐츠 요약
1. **정보**: 이름·이메일·전화·입사일·시급(매장기준/개별)·상태·푸시 보내기
2. **출퇴근**: 캘린더 미니 뷰 + 최근 7일 리스트 + 자세히 보기
3. **급여**: 월별 카드 리스트 (PAID/DRAFT 배지) + 명세서 진입
4. **연차**: 잔여 일수 + 신청 이력 (Phase 2: 승인 액션)

## 토큰
- 프로필 아바타: 80×80, 이니셜, `surfaceMuted` 배경, `brandPrimary` 글자
- 탭 indicator: 하단 2px line `brandPrimary` + 활성 텍스트 bold
- 메모 영역: `warning.bg` 배경 + 펜 이모지 노란 톤

## 상태
- 로딩: 스켈레톤 카드 3개
- 데이터 0: "아직 출퇴근 기록이 없어요" 일러스트
- 비활성 직원: 회색 톤 + "복구하기" CTA
- 시급 변경 모달: BottomSheet, 새 시급 입력 + 적용 시작일 (소급 X)

## API
- GET `/api/user/{id}`
- GET `/api/attendance/employee/{id}/monthly?year=...&month=...`
- GET `/api/payroll/employee/{employeeId}`
- GET `/api/wages/employee/{employeeId}/store/{storeId}`
- PUT `/api/wages/employee` (시급 변경)
- PUT `/api/user/{employeeId}` (정보 수정)
- DELETE 매장-직원 관계 비활성 (별도 API 필요 — P1)

## 엣지케이스
- 본인이 아닌 매장 직원 조회 시 403
- 시급 변경 시 변경 이력 자동 기록 (BE 측 처리 — P1)
- 사장 메모는 직원에게 절대 노출 X (response DTO 분리)
