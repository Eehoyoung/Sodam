# G-006 · 비밀번호 재설정 (3 step)

## 목적
영구 락아웃 방지. 이메일 → OTP → 새 비번 3단계 자동 진행.

## 페르소나·진입
- LoginScreen "비밀번호 찾기" 링크에서 진입
- 다음: 완료 시 LoginScreen 으로 자동 회귀 + 알림

## 레이아웃 (Step 1 - 이메일 입력)
```
┌─────────────────────────────┐
│ ← [뒤로]                     │
│                             │
│ 비밀번호 찾기                 │  (xxl bold)
│                             │
│ 가입하신 이메일을 알려주세요    │  (sm secondary)
│ 인증번호를 보내드릴게요         │
│                             │
│ ┌─────────────────────────┐ │
│ │ ✉ name@example.com      │ │  ← Input email type
│ └─────────────────────────┘ │
│                             │
│ [    인증번호 받기    ]      │  ← Primary lg fullWidth
│                             │
│ 로그인 화면으로 돌아가기 →     │  ← ghost link
└─────────────────────────────┘
```

## 레이아웃 (Step 3 - 새 비번)
```
┌─────────────────────────────┐
│ ← [뒤로]                     │
│                             │
│ 새 비밀번호 설정              │  (xxl bold)
│                             │
│ 안전한 비밀번호를 만들어 주세요  │  (sm secondary)
│                             │
│ ┌─────────────────────────┐ │
│ │ 🔒 새 비밀번호    [보기]   │ │  ← Input secure
│ └─────────────────────────┘ │
│ ▓▓▓▓░░░░ 강도: 보통           │  ← 강도 바 (3색)
│                             │
│ ┌─────────────────────────┐ │
│ │ 🔒 비밀번호 확인          │ │
│ └─────────────────────────┘ │
│                             │
│ ✓ 8자 이상                   │  (xs)
│ ✓ 대소문자 포함               │
│ ✓ 숫자 + 특수문자 포함         │
│                             │
│ [   비밀번호 변경하기   ]     │
└─────────────────────────────┘
```

## 토큰 사양
- OTP 박스: 1.5px border, 포커스 시 `brandPrimary` 강조 + scale 1.05
- 강도 바: 약함=`error`, 보통=`warning`, 강함=`success`
- 체크 항목: 미충족=`textTertiary`, 충족=`success` 색 변경
- 타이머: `monospace` (tabular-nums), `warning` 컬러

## 상태별 변화
- Step 1 발송 실패 → 다음 단계로 진행 X, inline error
- Step 2 OTP 오류 3회 시 5분 락아웃 (그래도 보안)
- Step 2 만료 → 모달 "인증번호가 만료되었어요. 다시 받으시겠어요?"
- Step 3 비번 정책 위반 → 체크 항목이 실시간으로 ✓/✗

## API
- POST `/api/auth/password-reset/request`  { email }
- POST `/api/auth/password-reset/verify`   { email, code } → { resetTicket }
- POST `/api/auth/password-reset/confirm`  { resetTicket, newPassword }

## 엣지케이스
- 이메일 미가입 시도 → BE 는 동일 응답 (account enumeration 방지). FE 는 "발송했다"로 표시
- 네트워크 끊김 → 토스트 + 재시도 버튼
- 백그라운드에서 1분 이상 지나면 Step 2 의 타이머 동기화 (Date.now() 기준)
