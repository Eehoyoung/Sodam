# S-301 · 급여 정산 플로우 (3 단계)

## 목적
월말 1회, 매장 전체 급여를 자동 계산 → 미세 조정 → 명세서 발급까지.

## Step 1 — 정산 기간·매장 선택
```
┌─────────────────────────────┐
│ ← 급여 정산                   │
│  ───────────────────────    │
│  1단계: 기간 설정              │
│                             │
│  매장                        │
│  ┌─────────────────────┐    │
│  │ 소담 데모 카페       ▼ │    │
│  └─────────────────────┘    │
│                             │
│  정산 기간                    │
│  [ 2026-05-01 ~ 05-31 ]    │  ← 기본: 이번 달 1일~말일
│                             │
│  근무 직원 수: 3명            │  (xs textTertiary)
│  총 근무시간: 245.5h          │
│                             │
│  [   다음: 미리보기   ]     │

```

## Step 3 — 확인 + 발급
```
┌─────────────────────────────┐
│ ← 3단계: 확인                  │  Progress 3/3
├─────────────────────────────┤
│ ✅ 정산 준비 완료              │  (lg success)
│                             │
│ 매장: 소담 데모 카페            │
│ 기간: 2026-05-01 ~ 05-31     │
│ 직원: 3명                    │
│ 총액: ₩2,358,400             │
│                             │
│ ☐ 직원에게 푸시 알림 발송       │
│ ☐ 명세서 PDF 자동 생성 (P1)   │
│                             │
│ [   명세서 발급하기   ]      │  ← 햅틱 medium
│                             │
│ 발급 후 24시간 이내 취소 가능   │  (xs textTertiary)
└─────────────────────────────┘
```

## 토큰
- Progress: 상단 3 칸, 현재=`brandPrimary`, 이전=`success`, 미래=`surfaceMuted`
- 직원 카드 펼침/접힘: 화살표 회전 애니메이션
- 가감 모달: BottomSheet, +/- 금액 + 사유 필수
- 발급 버튼: `tokens.shadow.brand` 강한 글로우

## 상태
- 미완 출퇴근 (퇴근 안 함) 직원 있음 → Step 1 차단 + 안내
- 가감 사유 비어있음 → Step 2 다음 비활성
- 발급 후 직원에게 푸시 자동 발송 (NotificationService.notifyPayrollPaid)

## API
- POST `/api/payroll/calculate` { storeId, startDate, endDate } → 미리보기
- POST `/api/payroll/{id}/adjustment` { amount, reason } (P1)
- POST `/api/payroll/confirm` 또는 PUT `/api/payroll/{id}/status?status=PAID`

## 엣지케이스
- 정산 중 새 출퇴근 기록 발생 → 경고 "정산 후 새 기록이 있어요"
- 이미 발급된 기간 재정산 시도 → 차단 + "기존 명세서 취소 후 재정산해 주세요"
