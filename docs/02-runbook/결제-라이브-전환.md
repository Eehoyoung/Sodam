# 결제 라이브 전환 런북 (key-ready)

> 코드는 전부 준비됨. **키 주입 + 모드 전환 + 마이그레이션 실행**만 하면 결제가 작동한다.
> 모든 항목은 인간 승인 사항(💳 결제 로직·🚀 배포·🔑 외부 키). 이 문서는 그 절차서다.

## 0. 사전 (외부 계약·승인)
- [ ] 토스페이먼츠 가맹 계약 + 실키 발급(client/secret/webhook secret)
- [ ] (단건결제 대리수취 시) 토스 **지급대행/분리정산 계약** + 법무 검토 — 세무사 지급 자동화 전 필수
- [ ] 약관·전자금융 동의·환불정책 문구 확정(변호사)

## 1. 백엔드 키 주입 (.env)
```
SODAM_INTEGRATION_TOSS_MODE=live        # mock → live
TOSS_CLIENT_KEY=live_ck_xxxxx
TOSS_SECRET_KEY=live_sk_xxxxx
TOSS_WEBHOOK_SECRET=whsec_xxxxx          # 토스 콘솔 웹훅 시크릿
```
- `mode=live` 인데 키가 비거나 `test_*` 면 **부팅 실패**(LiveTossBillingClient 가드) — 의도된 안전장치.
- 빈 webhook secret이면 웹훅 전량 거부(TossWebhookController). 정기결제 동기화 위해 권장.

## 2. 프런트 키 주입
```
SODAM_TOSS_CLIENT_KEY=live_ck_xxxxx      # FE env. isTossLive()=true 되어야 결제창 오픈
```
- `react-native-webview` 설치 필요: `npm i react-native-webview` (+ iOS `pod install`). 미설치 시 "결제 모듈 준비 중" 폴백.

## 3. DB 마이그레이션 (인간 승인 후 실행)
```
FLYWAY_ENABLED=true   # V1~V11 적용: 구독 티어 재편/청구주기/세무 단건주문 테이블
```
- ⚠️ 실행 전 **백업**. V10(구독 enum 리맵), V11(tax_service_order) 포함.

## 4. 동작 검증 (live 전 샌드박스 e2e)
- 정기결제: 구독 화면 → 결제창(빌링auth) → 첫 청구 DONE → 구독 ACTIVE.
- 단건결제: 세무 패키지 주문 생성 → 결제창 → confirm → 주문 PAID(매출=송객수수료, 예수금=세무사 전달분).
- 웹훅: 토스 콘솔에서 테스트 발송 → 서명 검증 통과 → PaymentHistory/구독 동기화.

## 5. 결제 로직 안전장치 (이미 코드에 반영)
- 멱등성: 동일 구독·청구년월 SUCCESS 1건이면 재청구 스킵(이중청구 차단).
- PAST_DUE 재시도 +3일 연장(무한 반복청구 차단).
- 단건결제 금액 위변조 방지: 서버 보관 주문 금액과 일치 검증.

## 하지 않는 것 (법적 라인 — 키로도 못 켬)
- ❌ 주민번호 저장 / 4대보험 **EDI 대행**(소담 명의 접수) = 개인정보보호법·공인노무사법 위반.
  → 4대보험은 **신고서 서식 자동작성 + 사장 직접 제출**만(주민번호 미저장).
- ❌ 세무·노무 **대행** = 세무사법·노무사법. → SW+**송객(레퍼럴)**만.
