## 5. SQLi·XSS·CSRF 결론
- **SQL 인젝션: 안전** — 전 `@Query` `:named` 파라미터 JPQL, nativeQuery 0, 사용자입력 연결 없음.
- **CSRF: 안전** — API 인증은 `Authorization: Bearer` 헤더만(쿠키 미인증). 미사용 JWT 쿠키도 Secure+SameSite=Strict 강화. (7d036c3)
- **XSS(FE WebView): 안전** — Toss customerKey=앱생성값, clientKey=config. originWhitelist는 카드사·3DS 외부 리다이렉트 필요로 '*' 유지(게이트=onShouldStartLoadWithRequest sentinel 검증).
- **FE 로깅**: api.ts 요청로그 `__DEV__` 가드(운영 비번 노출 차단), 정보화면 `Linking.openURL` http(s) 화이트리스트. (3040378)
