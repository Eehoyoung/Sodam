## 2. 화면별 명세

### S-001 · 홈 대시보드 (Owner Home)

**목적**: 오늘·이번 달의 상태를 1초 안에 파악.

**구성 위에서 아래로**:
1. 매장 전환 셀렉터 (Pill 형태, 매장 1개면 표시 안 함)
2. 오늘 출근 현황 카드 ("3/3명 출근 중", 미체크인 직원 빨강 배지)
3. 이번 달 누적 급여 카드 (예상 총액·근무시간 합계·잔여 영업일)
4. 액션 퀵그리드 (2x2)
   - "급여 정산하기" → S-301
   - "직원 추가" → S-501
   - "위치/반경 설정" → S-503
   - "노무·세무 팁" → S-701
5. 인사이트 배너 (예: "이번 주 야간 근무 5시간 ↑ 지난주 대비")
6. 광고 영역 (무료 플랜만 노출, AdMob)

**엣지케이스**:
- 매장 위치 미설정 → "출퇴근 인증을 시작하려면 위치를 설정해 주세요" 노란 배너
- 첫 진입 (직원 0명) → 빈 상태 일러스트 + "첫 직원 추가하기" CTA
- 오프라인 → 캐시 데이터 + "최근 업데이트: 12:34" 표시

**API**:
- GET `/api/auth/me`
- GET `/api/stores/master/current`
- GET `/api/store-queries/{storeId}/stats/attendance/today`
- GET `/api/store-queries/{storeId}/stats/payroll/month-to-date`

### S-101 · 출퇴근 보드 (Attendance Board)

**목적**: 현재 매장 직원들의 출근/근무/퇴근 상태를 한눈에.

**구성**:
1. 매장 선택 (다매장)
2. 날짜 셀렉터 (오늘 / 이번 주 / 이번 달 토글)
3. 직원 카드 리스트 — 각 카드:
   - 프로필(이니셜) + 이름
   - 상태 배지: `출근 대기` / `근무 중 03:24` / `퇴근 완료 8h 12m` / `미출근` / `결근`
   - 적용 시급 (이 직원의 매장 시급)
   - 우측 핸들: 수동 기록 추가 (메뉴)
4. 하단 액션: "수동 출퇴근 기록 추가" (예외 상황용)

**엣지케이스**:
- 운영시간 외 출근 → 카드 우상단 ⚠️ 아이콘 + 툴팁 "운영시간 외 기록"
- GPS 범위 밖 시도 → 별도 알림 + 사장에게 푸시 발생 (옵션)
- 동시 동일 직원 중복 출근 시도 → 차단 + 사유 로그
- 외부 매장 직원 (퇴직) → 회색 카드, 숨김 토글

**API**:
- GET `/api/attendance/store/{storeId}?date=YYYY-MM-DD`
- POST `/api/attendance/manual` (수동 기록)
- PATCH `/api/attendance/{id}` (시간 정정)

### S-201 · 직원 상세 (Owner View)

**목적**: 한 명의 직원에 대한 모든 정보·이력.

**탭**:
1. **정보**: 이름·이메일·전화·입사일·시급(매장기준/개별)·상태(활성/비활성)
2. **출퇴근**: 캘린더 뷰 + 일자별 상세
3. **급여**: 월별 누계 + 명세서 링크
4. **연차**: 잔여 일수 + 신청 이력

**액션**:
- 시급 변경 → S-202 (변경 이력 자동 기록)
- 매장에서 제외 → 비활성화 (Soft Delete, 30일 후 영구 삭제 알림)
- 푸시 보내기 (예: "오늘 출근 시간 확인해 주세요")

### S-301 · 급여 정산 (Payroll Run)

**목적**: 매월 25~말일 사이 1회 실행하는 핵심 액션.

**플로우**:
1. 정산 대상 매장·기간 선택 (기본: 이번 달 1~말일)
2. 자동 계산 결과 미리보기 (직원별 카드)
   - 기본/연장/야간/주휴 분리 표시
   - 사업소득세 3.3% 자동 차감
   - 비고 입력란 (선택)
3. 직원별 개별 조정 가능 (수동 가감 — 사유 필수)
4. "급여 명세서 생성" → 직원 앱에 푸시 + 본인 확인 요청
5. 송금 (외부 뱅킹 앱 호출 또는 토스 송금 링크 — Phase 2)
6. "지급 완료" 처리 → 상태 PAID

**엣지케이스**:
- 출근 기록이 없는 직원 → "0원 (근무 없음)" 안내
- 미완료 퇴근 기록 → "현재 근무 중 — 정산 전 종료해 주세요" 차단
- 주 15시간 미달 → 주휴수당 자동 0
- 환불/취소: 이미 PAID 명세서 → CANCEL 시 사유 + 직원 알림

**API**:
- POST `/api/payroll/calculate` (preview)
- POST `/api/payroll/confirm` (생성·푸시)
- PUT `/api/payroll/{id}/status?status=PAID`
- POST `/api/payroll/{id}/adjustment` (수동 가감)

### S-401 · 구독·결제 (Subscription)

**목적**: 플랜 가입/변경/해지.

**구성**:
1. 현재 구독 상태 카드 (플랜·청구주기·다음 결제일·카드 마스킹·일시정지 배지)
2. 플랜 카탈로그 (4종: FREE/STARTER ₩9,900/PRO ₩19,900/PREMIUM ₩39,900, S-402) + 청구주기(월/반년/연) 선택
3. 결제 이력 (S-403)
4. 일시정지/재개 · 해지 CTA
5. PRO 전용 기능 접근 시 페이월(402→업그레이드 유도)

**플로우 — 신규 가입**:
1. 플랜 선택 → S-402
2. "결제수단 등록" → 토스 SDK WebView (카드 등록)
3. authKey 획득 → POST `/api/billing/subscribe`
4. 성공 → 환영 모달 + 홈으로

**엣지케이스**:
- 첫 결제 실패 → 카드 재등록 페이지 + 에러 원인 안내
- 다음 결제 실패 → 3일 후 재시도, PAST_DUE 배지 노출
- 해지 후 환불 정책: 일할 환불 X (다음 결제일 전까지 ACTIVE 유지)
- 플랜 다운그레이드 → 즉시 변경 X, 다음 결제일에 적용

**S-501c 시급 설정**:

## 4. 상태 머신 — 사장님 관점 핵심 도메인

### 직원 관계 (EmployeeStoreRelation)
```
초대됨 → 활성 → 비활성 → (재활성 가능)
              ↘ 영구삭제 (30일 후)
```

### 정산 (Payroll)
```
DRAFT → APPROVED → PAID
   ↘ CANCELLED       ↘ REFUNDED (분쟁)
```

### 구독 (Subscription)
```
PENDING_PAYMENT → ACTIVE → CANCELLED → EXPIRED
                  ↕ PAUSED (비수기 일시정지 → 재개)
                     ↓
                  PAST_DUE → ACTIVE (재청구 성공, +3일 재시도)
                     ↓
                  EXPIRED (3회 실패)
```
