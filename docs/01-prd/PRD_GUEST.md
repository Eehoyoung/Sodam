# PRD-GUEST — 비회원·온보딩 명세

> 앱 첫 진입 ~ 가입 완료 직전까지의 화면.
> **핵심 가설**: 가입 전 3분 안에 "이 앱은 나에게 쓸만하다"를 느껴야 한다.

**버전**: 1.0
**작성일**: 2026-05-19

## 0. 비회원 페이지 KPI

| 지표 | 목표 |
|---|---|
| 앱 첫 실행 → 회원가입 시작 | 60% |

## 1. Splash → 온보딩 → 가입 흐름

### G-001 · Splash (1초)

- 소담 로고 + 슬로건 ("소상공인을 담다")
- 그라디언트 배경 (브랜드 오렌지 → 다크)
- 최대 1.5초, 부팅 길어지면 진행 표시

### G-002 · 온보딩 캐러셀 (3장)

**1장**: "출퇴근, NFC 한 번이면 끝"
**2장**: "급여, 자동으로 정확하게"
**3장**: "4대보험·세무 준비까지 한 앱에서" (※ "환급" 단정 카피 금지 — 사장 종소세는 구조적 비환급)

- 좌/우 스와이프, 점 인디케이터
- 하단 "건너뛰기" / "시작하기"
- 이미지: 일러스트(외주 디자인 의뢰 전까지는 SVG 플레이스홀더)

---

### G-003 · 역할 선택

- 카드 2개:
  - 🧑‍💼 "사장이에요" → MASTER 등급으로 가입
  - 🧑‍🎓 "알바·직원이에요" → EMPLOYEE 등급으로 가입
- 둘 중 하나를 선택해야 다음 화면 진행

---

### G-004 · 회원가입 방법

- 카카오 1탭 가입 (대표 CTA)
- 이메일 가입 (보조 CTA)
- "이미 가입했어요?" → G-005

---

### G-005 · 이메일 로그인

- 이메일 / 비밀번호
- 비밀번호 찾기 → G-006
- 비밀번호 표시/숨김 토글
- 자동 로그인 토글 (체크 시 Refresh Token 30일)

---

### G-006 · 비밀번호 재설정

- 이메일 입력 → OTP 6자리 (5분 유효)
- 새 비밀번호 입력 (정책: 8자 이상, 대소문자·숫자·특수문자 포함)
- 완료 시 자동 로그인

---

### G-007 · 이메일 회원가입 폼

- 이메일 / 비밀번호 / 비밀번호 확인 / 이름 / 전화번호(선택)
- 약관 동의 4종 (필수 3, 선택 1):
  - [필수] 만 14세 이상
  - [필수] 이용약관
  - [필수] 개인정보 처리방침
  - [선택] 마케팅 정보 수신 동의
- 이메일 중복 체크 (실시간, 디바운스 500ms)
- 비밀번호 강도 표시 (3색 바)

---

### G-008 · 카카오 로그인

- WebView (또는 시스템 브라우저) 카카오 OAuth
- 콜백 → 자동 회원가입 또는 로그인
- 카카오 이메일 미동의 시 안내 → 이메일 가입 안내

---

### G-009 · 가입 직후 첫 화면 (Welcome)

- 사장: "첫 매장을 등록해 볼까요?" → 매장 등록 화면
- 직원: "매장 코드를 받으셨나요?" → 매장 가입 화면 또는 둘러보기

## 2. 비회원도 접근 가능한 영역

- 정보 허브 (노무·세무·팁) — 로그인 없이 읽기
- FAQ
- 이용약관 / 개인정보 처리방침
- 회사 소개 (Phase 2: 랜딩 페이지)

비회원이 인증 필요 액션 시도 → 로그인 모달 슬라이드업.

---

## 3. 동의 화면 (필수)

각 약관은 풀텍스트 화면 + 동의 체크박스.
- [G-A1] 만 14세 이상 (KISA 권고)
- [G-A2] 이용약관 (Claude 초안 → 변호사 검토 — CONFIRM_REQUIRED.md C-8)
- [G-A3] 개인정보 처리방침
- [G-A4] (선택) 마케팅 정보 수신 동의

---

## 4. 에러·접근성

- 모든 폼 inline 에러 (필드 아래 빨강 텍스트)
- 키보드 회피(KeyboardAvoidingView) 자동 적용
- 색맹: 상태 표시는 색 + 아이콘 동시
- 폰트 크기 시스템 따름 (다이내믹 타입 지원)

---

## 5. 마케팅 진입 (딥링크)

URL 형식: `sodam://invite/{매장코드}`

- 사장이 매장코드 카카오톡 공유 → 직원이 링크 탭
- 미가입자 → 가입 후 자동으로 해당 매장 가입
- 가입자 → 매장 가입 신청 화면 직행
