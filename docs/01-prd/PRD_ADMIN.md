# PRD-ADMIN — 운영자(1인 사업가 본인) 백오피스 명세

> **이 문서의 위상**
> 1인 사업가가 출시 후 매일 켜는 운영 화면을 정의.
> 별도 운영자 앱은 안 만들고, **웹 백오피스 (Next.js or RN Web)** 로 가볍게.

**버전**: 1.0
**작성일**: 2026-05-19
**페르소나**: 본인(CEO/CTO/CFO 1인)

---

## 0. 운영자의 핵심 6대 작업

| # | 작업 | 빈도 |
|---|---|---|
| 1 | 신규 가입·결제 모니터링 | 매일 |
| 2 | CS 응대 (채널톡 자동, 에스컬레이션만) | 매일 |
| 3 | 콘텐츠(노무·세무·팁) 등록·수정 | 주 2~3회 |
| 4 | 결제 분쟁/환불 처리 | 주 1회 미만 |
| 5 | 에러·이상 트래픽 점검 (Sentry) | 매일 |
| 6 | 매출·MRR 대시보드 확인 | 매일 |

## 1. 운영자 백오피스 화면 (관리자 도메인 — `/admin`)

### O-001 · 운영 대시보드 (Dashboard)

**위에서 아래로**:
1. KPI 위젯 (오늘 신규/유료 전환/이탈/MRR)
2. 최근 7일 가입 트렌드 (그래프)
3. 미응대 CS 카운터 (배지)
4. Sentry 이슈 트렌드 (상위 5)
5. 결제 실패 누적 (PAST_DUE 구독 수)

---

### O-101 · 사용자 관리

- 사용자 검색 (이메일/이름/ID)
- 사용자 상세:
  - 등급 변경 (Personal ↔ MASTER ↔ EMPLOYEE)
  - 강제 비활성화 (위반 시)
  - 구독·결제 이력
  - 디바이스 토큰 목록
  - 출퇴근/급여 데이터(읽기 전용)
  - 비밀번호 강제 재설정 메일
- 일괄 작업: CSV 가져오기/내보내기 (Phase 2)

---

### O-201 · 매장 관리

- 모든 매장 리스트(검색·필터)
- 매장 상태(활성/소프트삭제) 토글
- 사업자번호 확인 (NTS API — Phase 2)
- 비정상 매장 표시 (반경 0, 직원 0 등)

---

### O-301 · 결제 운영

- 구독 리스트 (상태 필터)
- 결제 이력 (성공/실패/환불)
- 분쟁 처리: 환불 실행(토스 cancel API 호출)
- 웹훅 수신 로그 (PaymentHistory 기반)
- PAST_DUE 사용자 일괄 알림 발송

**중요**: 환불 실행은 운영자 비밀번호 재인증 필수.

---

### O-401 · 콘텐츠 관리 (CMS)

- 카테고리: 노무 / 세무 / 정책 / 팁 / FAQ
- 마크다운 에디터 (이미지 업로드 S3)
- 게시 상태(초안/공개/숨김)
- 노출 우선순위
- 조회수 표시

---

### O-501 · Q&A 모더레이션

- 신규 질문 큐 (사장 / 직원 분리)
- 답변 작성 → 사용자에게 푸시
- 부적절한 질문 숨기기
- 자주 묻는 질문은 FAQ로 변환

---

### O-601 · 푸시 발송 (마케팅)

- 세그먼트 선택 (전체 / 사장 / 직원 / 무료플랜 / 유료플랜)
- 메시지 작성 + A/B 테스트 (Phase 2)
- 발송 예약
- 발송 후 결과(도달/오픈) 리포트

---

### O-701 · 데이터/통계

- 일/주/월별 KPI 집계
- 매출·MRR 차트
- Funnel: 가입 → 매장등록 → 출근1회 → 유료전환
- Cohort 잔존율 (Phase 2)
- CSV 익스포트

---

### O-801 · 시스템 설정

- 환경 변수 보기/마스킹 (편집은 AWS Secrets Manager 직접)
- 캐시 무효화
- 배치 수동 실행 (BillingScheduler 수동 트리거 등)
- 점검 모드 토글

---

### O-901 · Phase 2 — 세무사 파트너 콘솔

- 세무사·노무사 파트너 계정 관리
- 송객 케이스 큐 (세무 패키지 주문 = `TaxServiceOrder`)
- 대리수취 정산 내역 (예수금=세무사 전달분 / 매출=송객수수료) · 월 net 정산
- 파트너 KPI (처리 시간, 송객 전환율) — ※ 환급 정률 폐기

---

## 2. 운영자 권한 (Admin Auth)

- 관리자는 별도 화이트리스트 이메일만 가능 (`ADMIN_EMAILS=...`)
- `@Secured("ROLE_ADMIN")` 모든 admin API에 적용
- 2FA TOTP 권장 (Phase 2)
- 운영자 행동 로그 (감사 — AuditLog 테이블, Phase 2)

---

## 3. 운영 자동화 (Claude/AI 에이전트 활용)

다음은 사람 없이 자동 처리:

| 시나리오 | 자동화 방법 |
|---|---|
| 결제 실패 알림 | BillingScheduler + NotificationService |
| 출퇴근 누락 알림 | AttendanceMissingScheduler (Phase 2) |
| 채널톡 1차 응대 | 챗봇 시나리오 5종 |
| Sentry 이슈 분류 | Sentry 룰 (level + tag) |
| FAQ 자동 응답 | 미리 정의된 키워드 매핑 |
| 콘텐츠 SEO 최적화 | 운영자 ↔ Claude 협업 |
