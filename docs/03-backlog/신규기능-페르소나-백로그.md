# 3차 — 구현 규격: 코드 규칙 + FE 디자인 목업/와이어프레임 (2026-06-16)

> 위 백로그를 실제로 만들 때 따르는 **코드 컨벤션**과, FE 화면이 필요한 항목의 **토스식 와이어프레임**.
> 모든 목업은 DS v3 토큰·컴포넌트 기준(좁은 박스 = 모바일 1열, "한 화면=한 가지").

## A. 공통 코드 규칙

### BE (Spring Boot / Java 17)
- **Controller**: 권한 가드 패턴 준수 — 사장 API는 `@MasterOnly` + `storeAccessGuard.assertMasterOwnsStore(...)`, 직원 API는 `assertEmployeeInStore(employeeId, storeId)`. 본인 리소스는 토큰 주체로만 조회(`/my` 패턴), 파라미터로 타인 id 받지 말 것.
- **DTO**: `record` + 정적 `from(Entity)`. **직원 노출 금지 필드 절대 제외** — `WageHistory.changedBy`, `EmployeeStoreRelation.ownerMemo`(엔티티 주석 "직원에게 절대 노출 X")는 직원 응답 DTO에서 빼고 별도 DTO 분리.
- **Service**: `interface` + `Impl`, `@RequiredArgsConstructor`, 쓰기엔 `@Transactional`. 비즈니스 로직은 컨트롤러에 두지 않음.
- **상수 분리**: 법령 변동 대비 — 최저임금·세율표·신고기한·연소근로 한도(§69)·연장 한도(§53)는 `core/.../constant`에 별도 파일. 직접 하드코딩 금지.
- **시간**: 시각 의존 로직은 `Clock` 주입(테스트 고정). 스케줄러는 `AttendanceMissingScheduler` 패턴 재사용.
- **멱등**: 보상·발급·결제성 카운터(레퍼럴 무료월, 월1회 무료발급)는 멱등키 필수(`매장+년월`, 이미 처리 시 무시).
- **면책**: 모든 급여·세무·연차 산출물은 기존 `PAYROLL_DISCLAIMER` 패턴 재사용 — "참고용 추정, 실제와 다를 수 있어요".
- **PII Hard No**: 주민번호·계좌·가족관계증명서 **원본 미저장**. 확인 플래그/메타만. 지급명세서는 이름+내부ID까지, 주민번호는 홈택스에서 사장이 보완.
- **테스트**: 급여/출퇴근 80%·시급/Auth 70%. 신규 노무·세무 계산은 경계값(주 15h 주휴, 만 18세 경계, 52h 한도) 케이스 필수.
- **마이그레이션**: Flyway 스크립트 **작성 OK, 실행은 인간 승인**. 신규 컬럼(`payday`,`taxMode`,`weeklyHolidayEligible`,`taxType`)·신규 엔티티(`WorkShift`,`EmployeeDocument`,`DomainEvent`)는 nullable 기본 + 백필 전략 명시.

### FE (React Native 0.81 / TS) — DS v3 / 토스식
- **컴포넌트는 DS만**: `ScreenContainer`(여백 24/28, footer CTA 슬롯) · `AppButton`(h56·radius18·primary 1개) · `AppCard`(radius20·그림자) · `HeroNumber`(숫자 40~56px 히어로) · `StepScaffold`(진행형 스텝) · `AppText`/`AmountText` · `AppBadge` · `SegmentedControl` · `EmptyState`/`SuccessState`. 신규 RN 프리미티브 직접 스타일링 지양.
- **레이아웃 원칙**: 한 화면=한 가지 · 숫자 히어로 1개 · **하단 풀폭 CTA 1개** · 압도적 여백(24~40) · 진행형 단계는 점진 공개(progressive disclosure).
- **색**: 하드코딩 hex 금지 → `useThemeColors()`(다크모드), 간격 `tokens.spacing`.
- **아이콘**: `Ionicons`만, **이모지 UI 금지**(DS v3). 본 목업의 `[아이콘]`은 Ionicons 라인 아이콘으로 치환.
- **금액·이름**: `numberOfLines`/`adjustsFontSizeToFit`로 픽셀 깨짐 방지(긴 매장명·큰 금액).
- **타입**: `any` 금지, `null` 안전 명시.
- **카피**: 사장/직원 모두 친근한 존댓말("사장님, ~했어요" / "~해 주세요"). 추정 수치엔 면책 1줄 동반.

- **A3 고용세액공제·B7 두루누리**: 사장 홈 `AppCard` 신호 카드("지원 가능 직원 2명") → 상세는 A2 세무자료 패턴.

## C. 구현 순서 내 디자인 의존성
1. **HeroNumber·StepScaffold·SuccessState는 이미 DS v3에 존재** → 위 목업 전부 신규 프리미티브 없이 조립 가능.

3. 나머지는 `AppCard`+`HeroNumber`+`AppBadge`+풀폭 CTA 조합으로 끝 — 디자인 리스크 낮음.
