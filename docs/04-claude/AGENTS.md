# AGENTS.md — 소담 프로젝트 서브에이전트 정의

이 파일은 Claude Code의 **서브에이전트 활용 가이드**다.
1인 사업가가 출시까지 가는 동안, 다음 에이전트들이 도메인 전문가 역할을 대행한다.

> **사용 원칙**
> - 본 컨텍스트가 더 효율적이면 호출하지 않는다 (불필요한 비용 회피).
> - 병렬 가능한 작업은 동시 호출.
> - 각 에이전트는 **결과만 반환**, 의사결정은 메인 Claude/사용자가 한다.

---

## 🤖 도메인 에이전트 카탈로그

### 1. `payroll-validator` — 급여 계산 정합성 검토

**언제 호출**
- 급여 계산 로직(`PayrollService`, `Payroll.java`)을 수정한 직후
- 노동법 개정 시 영향 분석
- 급여 명세서 표시 오류 신고 시

**입력**
- 변경된 파일 경로
- 검증할 케이스 (시간외/야간/주휴/감급 등)

**산출**
- 한국 노동법 기준 정합성 OK/NG
- 어긋난 케이스의 구체적 수치 비교
- 단위테스트 추가 권장 항목

**호출 방식**
```
Agent(
  description="급여 로직 정합성 검토",
  subagent_type="general-purpose",
  prompt="다음 케이스로 backend/src/main/java/com/rich/sodam/service/PayrollService.java 의 calculate 메서드를 검증해주세요: ..."
)
```

**산출**

### 3. `security-auditor` — 보안·개인정보 점검

**언제 호출**
- 새 API 엔드포인트 추가 시
- 사용자 입력 받는 화면 추가 시
- 분기 1회 정기 점검 (Master Plan §2.5)

**입력**
- 변경된 파일/엔드포인트
- 데이터 흐름 (입력 → 저장 → 출력)

**산출**
- OWASP Top 10 기반 취약점
- PII 노출 가능성
- 마스킹/암호화 권장 사항

### 8. `sodam-designer` — 디자인 시스템·UI 일관성 가디언

**언제 호출**
- 신규 화면(.tsx) 추가/수정 시 디자인 토큰 적용 검토
- 디자인 일관성 회귀 의심 시 (분기 1회 정기 + 출시 전 1회)
- 색·간격·타이포·그림자 등 시각 결정 필요 시
- 브랜드 톤(따뜻한 오렌지 #FF6B35) 위반 의심 시
- 새 컴포넌트 추가 전 기존 공용 컴포넌트(Button/Card/Input/Badge/Header)로 해결 가능한지 판단
- 접근성(WCAG AA) 컬러 대비·터치 영역(44pt) 점검
- 다크모드·시스템 폰트 동적 크기 대응 검토 (Phase 2+)

**입력**
- 변경된 .tsx 파일 경로 또는 디자인 결정이 필요한 시나리오
- (선택) 기존 화면과의 비교 대상

**산출**
1. 디자인 토큰(`src/theme/tokens.ts`) 위반 항목 목록 (file:line + 권장 토큰 키)
2. 하드코딩 색상(#xxxxxx)·간격·폰트 사이즈 검출 + 대체 토큰 제안
3. 공용 컴포넌트 재사용 가능 항목 (커스텀 View 대신 Card 권장 등)
4. 접근성 위반 (대비 < 4.5 / 터치 영역 < 44pt) 보고
5. 통일성 개선 후 미리보기 코드 패치 제안

**검토 체크리스트** (에이전트 호출 시 자동 적용)
- [ ] 색상이 `tokens.colors.*` 에서 import 되었나
- [ ] 간격이 `tokens.spacing.*` 사용 (마법의 숫자 X)
- [ ] 폰트 크기/굵기 `tokens.typography.*`
- [ ] 라운드 `tokens.radius.*`
- [ ] 그림자 `tokens.shadow.*`
- [ ] 브랜드 그라디언트는 `tokens.gradient.brand` 사용 (직접 colors 배열 X)
- [ ] 텍스트 톤이 사장님/직원 페르소나에 맞는가 (다정한 존댓말)
- [ ] 금지어 ("혁신/AI 기반/스마트") 미사용
- [ ] 빈 상태 일러스트는 다정한 한 줄 카피 + 명확 CTA 포함
- [ ] 에러 메시지가 사용자가 다음 액션을 알 수 있게 작성됨

**브랜드 헌법 (이 에이전트가 따르는 절대 원칙)**
1. **#FF6B35** — Primary, 모든 핵심 CTA · 활성 상태 표시
2. **#1C1917 / #57534E** — Text 위계, 본문은 Primary 만 Secondary 보조
3. **warm gray** (`surface`, `surfaceMuted`) — 차가운 회색 금지
4. **터치 영역 44pt+** — 작은 아이콘은 padding 으로 확보
5. **그림자는 절제** — `shadow.md` 가 기본, `shadow.lg` 는 모달/오버레이만
6. **모서리 둥글기** — 카드 12pt(`radius.lg`), 버튼 8~16pt
7. **그라디언트** — 사장 환영·직원 CTA 동그라미 한정 (남발 X)
8. **폰트 시스템 우선** — 별도 폰트 번들 X (앱 크기/부팅 최적화)

**호출 예**
```
Agent({
  description: "신규 화면 디자인 일관성 검토",
  subagent_type: "general-purpose",
  prompt: "src/features/owner/screens/EmployeeDetailScreen.tsx 를 sodam-designer 룰로 검토. \
           tokens 위반·하드코딩·접근성·공용 컴포넌트 재사용 가능성 보고. \
           수정 패치 제안 포함. 50줄 이내."
})
```

**병렬 호출 권장**
- 신규 화면 PR 직전: `sodam-designer` + `korean-content-tone-checker` 동시 호출
- 출시 직전 점검: `sodam-designer` + `play-store-reviewer` + `security-auditor` 동시

## ❌ 서브에이전트를 쓰지 않을 때

- 단순 코드 편집·리팩토링
- 명확한 버그 수정
- 문서 작성
- 1~2개 파일 검색

이 경우 메인 Claude가 직접 처리한다.

## 🧠 메모리 (Claude memory) 연동

장기 컨텍스트는 `.claude/projects/.../memory/` 에 저장.
서브에이전트는 메모리를 직접 읽지 못하므로, 호출 시 필요한 컨텍스트를 **명시적으로 프롬프트에 포함**.

자주 참조되는 정보(매장 도메인 룰, 한국 노동법 상수 등)는 메모리로 저장해 메인 Claude가 매번 같은 컨텍스트를 재생산하지 않도록 한다.
