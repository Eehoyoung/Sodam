# 소담(SODAM) — 프로젝트 정체성 & 방향성

> **소상공인이 만든, 소상공인을 담는 단 하나의 운영 비서**
> 1인 사업가가 출시·운영하는 모바일 우선(B2B) SaaS 프로덕트.

---

## 1. 한 줄 정의 (Elevator Pitch)

> **"직원 1~5명짜리 가게의 출퇴근·급여·세무를 자동으로 굴려주는, 사장님의 주머니 속 운영팀."**

- **제품명**: 소담 (SODAM) — "**소**상공인을 **담**다"
- **카테고리**: SMB Workforce & Tax Ops (근태 + 급여 + 세무 자동화)
- **플랫폼**: React Native (Android 우선 → iOS → Web) + Spring Boot API
- **타겟 매출 도달 시점**: 출시 후 12개월 이내 MRR ₩5,000,000 (구독 300건 + 환급 수수료)

---

## 2. 누구를 위한 제품인가 (ICP — Ideal Customer Profile)

### Primary
- **카페·음식점·편의점·미용실·소형 학원**을 운영하는 **사장 1명 + 알바/직원 1~5명** 규모의 소상공인
- 연 매출 1억 미만, 사장이 직접 회계/노무를 처리해야 하는 **"내가 곧 인사팀이자 회계팀"** 사업주
- 종합소득세·4대보험·주휴수당이 매년 헷갈리고 **세무사 고용은 부담**스러운 단계

### Secondary
- **알바생 본인**: 출퇴근 인증, 급여 명세서, 정산 내역을 확인하고 싶은 직원
- **2~3개 매장을 운영하는 멀티 오너**(MasterStoreRelation 도메인 이미 존재)

### Anti-Persona (의도적으로 무시)
- 직원 20명+ 중견 사업장 → 시프티/알밤/플렉스에 양보
- 대형 프랜차이즈 본사 → 자체 ERP 사용
- 무인매장 only → 출퇴근 수요 없음

## 3. 정체성의 3대 축 (Brand Pillars)

| 축 | 의미 | 외부 메시지 |
|---|---|---|
| **현장성 (Authentic)** | 소상공인이 직접 기획·개발 | "사장님 마음을 아는 앱" |
| **자동화 (Automatic)** | 출퇴근 → 급여 → 세무가 한 줄로 연결 | "복붙·엑셀·계산기 0" |
| **부담 없음 (Accessible)** | 무료 진입 + 쓴 만큼 과금 | "월 1.5만 원이면 노무사·세무사 비용 절감" |

---

## 4. 차별점 — 왜 시프티/알밤이 아니라 소담인가

| 경쟁사 | 강점 | 약점 (소담의 기회) |
|---|---|---|
| 시프티(Shiftee) | 근태관리 1위, UI 깔끔 | 세무 연계 없음, 직원 10명+ 타겟 |
| 알밤 | 무료 출퇴근 | 급여 계산 약함, 세무 없음 |
| 자비스/삼쩜삼 | 세무 환급 강자 | 근태/급여 없음 |
| **소담** | **근태(NFC+GPS) + 급여 자동 + 종소세 환급 + 노무 콘텐츠가 하나의 앱** | (1인 사업가 운영이 약점이자 강점) |

### 핵심 기술적 차별점
1. **NFC + GPS 이중 인증**: 부정 출퇴근을 막는 Haversine 반경 검증 (이미 `Store.isUserInRadius` 구현됨)
2. **주휴수당 자동 계산**: 한국 노동법에 맞춘 주 15시간 룰 (이미 `Attendance.weeklyAllowance` 필드 존재)
3. **세무사 연결 환급 모델**: 직접 노무사·세무사를 고용하지 않고 **파트너 환급 수수료 분배**

## 5. 6개월·12개월 방향성

### Phase 1 — MVP 출시 (M0 ~ M3)
- 무료 플랜 + 비즈니스 플랜(월 15,000원)만 출시
- Android 단독 출시 → Play Store 비공개 베타 → 정식 출시
- 목표: 활성 사업장 100개 / 무료 → 유료 전환 5%

### Phase 2 — 수익화 (M4 ~ M8)
- 환급형 플랜 추가 (종합소득세 환급 수수료 10~20%)
- 세무사 파트너십 1~2곳 확보
- iOS 출시
- 목표: MRR ₩1,500,000 / 환급 수수료 누적 ₩3,000,000

### Phase 3 — 확장 (M9 ~ M12)
- 프리미엄 플랜(월 50,000원, 세무사 1:1) 정식화
- 다매장 사업주 기능 강화
- 광고 매출(무료 플랜) 시작
- 목표: MRR ₩5,000,000

## 6. 비기능 정체성 (어떻게 만들 것인가)

- **모바일 우선**: 사장님은 PC를 안 켠다. 모든 액션은 폰에서 끝나야 한다.
- **한국어 우선**: 영어/다국어는 Phase 3 이후.
- **개인정보 보호**: 주민번호·계좌번호는 저장하지 않거나 암호화. 4대보험 신고는 정부24 링크로 위임.
- **느린 인터넷 가정**: 매장 와이파이가 끊겨도 출퇴근 기록은 로컬 큐 후 동기화.
- **혼자 운영 가능한 스택**: AWS 단일 리전, RDS + Redis + EC2/ECS 1대로 100사업장은 커버.

## 7. 핵심 가치 선언 (Value Proposition Statement)

> 직원 1~5명짜리 가게의 사장에게,
> 소담은 **출퇴근부터 종합소득세 환급까지 한 앱에서 끝내는 자동 운영 비서**다.
> 시프티는 근태만, 자비스는 세무만 처리하지만,
> 소담은 **사장이 매일 직접 굴리는 운영 흐름 전체를 자동화**한다는 점이 다르다.

---

## 8. 안 할 것 (Non-Goals)

명확하게 거절해야 손이 안 늘어진다.

- ❌ POS·매출 관리 — 캐치테이블/도도포인트 영역
- ❌ 재고·발주 — 사장의 페인이 아님
- ❌ 직원 채용/구인 — 알바몬 영역, 1인이 못함
- ❌ 다국어 — Phase 3 이후
- ❌ 데스크탑 전용 기능 — 모바일에서 안 되는 건 만들지 않음
