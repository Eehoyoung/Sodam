# SODAM 수익화·수익모델 — 확정안 (v3.0)

**개정일**: 2026-06-06 (v2.0 → v3.0)
**의결**: 6인 페르소나 토론 — CFO·CEO·소상공인 사장님·세무사 + **공인노무사·VC 심사역·그로스 PM·CTO** + 메인(위원장) 판정
**지위**: PRD §5(수익 모델) 대체·확정. 단위는 **원**(k·M 약어 미사용), **파생 지표는 천원 미만 절사**(가격 끝자리 9는 의도적 마케팅값으로 유지).
**전제**: 약관·면책·정기결제 고지 법률 문구, 결제 로직, **세무사법·공인노무사법 유권해석·세무/노무사 계약·토스 지급대행 계약**은 인간/전문가 승인 사항.

## 4. 세무·노무 수익 — 송객 한정, 대행 안 함 (세무사법·노무사법 안전선)

### 4-1. 세무: 정액 종소세 신고 송객(환급 정률 폐기 유지) + 고용세액공제 + 반복세무
- **고객 99,000~149,000원/건**(시장가 100,000~200,000원 하회). 환급 0원인 사장도 "신고는 해야" 구매 → 환급 의존 제거(세무사: 사장 종소세는 구조적 비환급).
- **SODAM = 데이터 정리·증빙·송객(SW)만.** 신고·계산·환급액 확정은 100% 세무사. 시뮬레이터는 "참고용·세무사 검토 전" 워터마크.
- **진짜 데이터 해자 = 통합고용세액공제**(상시근로자 수 월별 증빙·사후관리 추징 방어). "환급"이 아니라 "직원 쓰는 사장 절세".
- **반복세무(부가세 분기·원천세 매월)가 진짜 LTV** — 종소세(연1회)는 시즌 캠페인.

### 4-2. 노무: 4대보험·노동법은 "준비"까지만, 대행은 노무사 송객
- **4대보험 "신고서 자동작성 + 사장 직접 제출"** — 소담 명의 접수·API 대행 금지(공인노무사법·보험사무대행 위반). 진짜 대행은 **노무사 송객(PREMIUM)**.
- **노동법 경고 면책 3중**: ① UI에 "입력값 기준 추정치·실제는 근로계약/실태에 따라 상이" 명기(약관만으론 불충분) ② 입력 로그 보존 ③ **엔진 버그는 면책 불가** → 노동법 상수 분리 + 법개정 갱신 SLA.
- **신규 수익**: 노무사 송객(취업규칙·진정·근로감독 대응) + **전자 근로계약서 서명·임금대장/명세서 3년 법정보관**(SW 영역, 자격 무관, 해자 강화).

### 4-3. 도입 게이트 (M6 선결 — 순서)
1. **세무사법·공인노무사법 유권해석**(어디까지 SW) — 계약보다 먼저.
2. **주민번호 충돌 해소**(4대보험 신고엔 주민번호 필수 vs CLAUDE.md 금칙) — 미보관·즉시폐기·고유식별정보 별도동의·암호화 설계.
3. 세무사·노무사 파트너 계약 ≥ 1건.
4. 정산·송객·**단건결제** 코드 E2E + 환불·책임·약관 확정.
> 통과 전 세무·노무 송객 매출 인식 0. 구독만으로 BEP 검증.

## 5. 💳 세무파트너 비용 구조 + 종소세 기장료 납부방법 (CTO 설계)

### 5-1. 세무파트너 비용 (송객 1건당, 원 단위)
| 항목 | 금액 |
|---|---|
| 고객 수취(정액 패키지) | 99,000~149,000원 |
| 세무사 지급(손익분기 충족) | 70,000~100,000원 |
| 소담 명목 마진 | 30,000~49,000원 |
| (−) 부가세 처리 | **대리수취 구조로 송객수수료만 과세**(재판매 시 부가세 10%로 마진 반토막) |
| (−) CS(자료 누락·일정 조율) | 건당 5,000~10,000원 상당(1인 운영 시간) |
| (−) 세금계산서/정산(팝빌·바로빌) | 건당 1,000~2,000원 |
| (−) 환불 충당(5% 가정) | 건당 2,000~3,000원 |
| **= 실질 순마진** | **건당 15,000~30,000원** |
- **정산: 월 1회 net**(세무사 지급 − 당월 환불분, 익월 5영업일). **세무사 선지급 절대 금지**(환불 회수 불가).

### 5-2. ★ 종소세 기장료 납부방법 — 확정 권고
4안 비교 후 **권고: ① 토스 단건결제 즉시수금 + ④ 대리수취(에스크로) 회계 = 하이브리드.**

| 방식 | 미수금 | 부가세 | 세무사법 | 판정 |
|---|---|---|---|---|
| ① 앱내 PG 단건결제 | 낮음 | 재판매 시 부담 | 재판매면 위험 | 수금방식으로 채택 |
| ② 구독 분할 청구 | 높음(중도해지 잔액) | 혼재 | 모호 | ✗ |
| ③ 환급액 차감 | 매우 높음(자금 수개월 묶임) | — | 성공보수 리스크 | ✗(국세청 연동 없음) |
| ④ 세무사 직접 청구 | 0 | 수수료만 | 가장 안전 | 회계구조로 채택 |

**확정 흐름**: 고객이 **토스 단건결제**로 패키지 결제(수금 즉시·미수금 0) → 소담은 이를 **재판매 매출이 아니라 "세무사 전달 예수금"으로 처리**, **소담 매출은 송객수수료(30,000~49,000원)만 인식**(부가세 베이스 축소) → 세무사에 월 net 지급. **선결 조건**: 토스 **지급대행/분리정산 별도 계약** + 법무 검토(외부 계약 = 인간 승인). 미싱 코드 = **일회성 PG 결제 플로우 1개**(현재 빌링키 청구만 존재, 신규 3~5일).

## 7. GTM·성장 레버 (v3, 우선순위)

1. **activation 재설계(최우선, 매출 모델보다 먼저)**: aha를 **D0 사장 단독**("자기 자신을 첫 직원으로 등록 → 급여 미리보기에서 주휴 포함 금액 확인")으로. 직원 초대·출퇴근(2인 협응)은 그 다음 단계로 분리.
2. **페이월 1순위 = 월급날 명세서 발급**(법정 의무·시점 명확·가치 직후). 4대보험·인건비비율은 **PRO 업셀** 트리거.
3. **사장→사장 레퍼럴**(그로스PM: 워터마크는 알바를 모음, 사장 아님) — 사장 커뮤니티·소개 인센티브에 투자.
4. **세무사·노무사 화이트라벨(B2B2B)**: 파트너가 자기 고객에게 SODAM(SW) 배포 → 소담은 클린 SaaS.
5. **A/B 우선순위**: ① FREE 명세서 "월1회 실발급" ② STARTER 4대보험 맛보기→PRO ③ FREE 직원 2 vs 3 ④ STARTER 9,900 vs 12,900.

## 부록 — 토론 경과 (감사 추적)

- **R1~R2 (CFO·CEO)**: 환급 MRR 제외·광고 폐기·LTV 13개월·12M MRR 2,500,000원.
- **R3 (사장님·세무사, v2)**: 직원수 과금→숨김 / 4~7명 빈구간 / **환급 정률 폐기**(비환급성·세무사 몫 음수·세무사법) → 정액 신고·고용공제·반복세무.
- **R4 (노무사·VC·그로스PM·CTO, v3)**:
  - 노무사: 4대보험 대행 위법·주민번호 충돌(차단)·퇴직금/연차/근로계약서 누락·노무사 송객.
  - VC: 보수 비정합·Lifestyle·생계 런웨이·세무는 옵션.
  - 그로스PM: 바텀업 M12 MRR 150,000~884,000원·activation 절벽·사장 레퍼럴.
  - CTO: 정기결제 기구현·COGS 300~1,200원·기장료 단건결제+대리수취·결제버그 2건.
- **R5 (메인 판정)**: 매출 재작성, 4대보험 송객화, 주민번호 게이트, 노무기능 편입, FREE 균형 A/B, 기장료 결제 확정, 결제버그 필수수정, Lifestyle 수용.

**한 줄 결론**: SODAM은 **부트스트랩 Lifestyle SaaS**다(VC). 진짜 병목은 가격이 아니라 **activation**이며(그로스PM), 세무·노무는 **대행이 아닌 송객으로만** 적법하게 수익화하고(세무사·노무사), 결제·정산은 **단건결제+대리수취**로 깔되 **결제 버그부터 막아야** 한다(CTO). 그 위에서 **"직원 쓰는 사장님의 실근로데이터→정확→분쟁 증거력"**이 유일한 해자다.
