## 0. 결론 한 장 (Executive Summary)

> **"좁은 시장(직원 1~5명)에서는 박리다매가 수학적으로 불가능하다. 낮은 진입 마찰로 사용자를 들이되, 게이팅·업셀 사다리로 ARPU를 끌어올리고, 세무 환급은 '검증된 뒤' 얹는 업사이드로 둔다."**

**확정 4티어 + 애드온** (월정액, 연납=2개월 무료):

| 티어 | 월정액 | 연납(÷12 실질) | 포지션 |
|---|---|---|---|
| **FREE** | ₩0 (광고 없음) | — | 체험·획득 깔때기 (수익원 아님) |
| **STARTER** | **₩9,900** | ₩99,000 (₩8,250) | 진입 — 1인 가게 급여 자동화 |
| **PRO** | **₩19,900** | ₩199,000 (₩16,580) | **본가(anchor)** — 풀 급여+노동법+환급연동 |
| **PREMIUM** | **₩39,900** | ₩399,000 (₩33,250) | 다매장·세무사 1:1 |
| **멀티매장 애드온** | **+₩9,900/매장** | — | PRO 이상에서 2호점부터 |

**환급형**: 구독 매출(MRR)에서 **100% 제외**. M6 게이트(세무사 계약 + 정산코드 E2E + 환불정책) 통과 후 M7~ PRO 연납에 **이연수익**으로 편입. 수수료 **환급액 15%(min ₩30,000 / max ₩100,000)**, 구독자 우대.

**목표 재설정**: 회사 단일 목표 = **구독 MRR**. 환급은 별도 GMV KPI.
- 12개월: 구독 MRR **₩2.5~3.0M** (현금 BEP 초과, 자립). 기존 PRD "₩5M"은 환급 포함·낙관 환상치로 **폐기**.
- 18개월: 구독 MRR ₩5M (환급 업사이드 가속 시).

## 3. 단위경제·BEP (토론 합의 수치)

**블렌디드 ARPU**(믹스 STARTER 50%·PRO 35%·PREMIUM 15% + 멀티매장 일부) ≈ **₩16,000~18,900/월**.
**그로스마진**: ARPU − 변동비(PG 3% + CS잠식 + SMS/지오코딩) ≈ **₩15,500/유료/월 (마진 ~82%)**.

| 지표 | 값 | 비고 |
|---|---|---|
| **현금 BEP**(창업자급여 제외) | **유료 ~65명** | 월 고정비 ₩1.0M ÷ ₩15,500. **M8~9 달성 가능** = 운영 자립 |
| **완전 BEP**(창업자급여 ₩3.5M 포함) | **유료 ~290명** | 12개월 내 미달, 18개월 트랙 |

| CAC | ₩30,000(낙관)~₩50,000(현실) | LTV/CAC = **4.0~6.7** ✅ |

## 4. 환급형(COMMISSION) — 2단계 게이트 (재무 규율)

> 환급은 **검증 전까지 매출 0**. "미인도 약속에 선결제"는 캐시플로우 개선이 아니라 환불 부채다.

**1단계 (M0~M6): 매출 인식 0.** 환급은 PRO의 "부가 베네핏"으로 마케팅만. BEP는 구독만으로 검증.

**게이트 통과 조건 (M6 시점 AND):**
- [ ] 세무사 파트너 계약 **≥ 1건 서명**
- [ ] 환급 정산 코드 **E2E 1건 실제 처리**(현재 코드 0%)
- [ ] **환불 정책·예치금 회계처리 확정**(법률 문구 = 인간 승인)

**2단계 (M7~): PRO 연납 번들 편입.** 수수료 **환급액 15%(min ₩30,000/max ₩100,000)**, 구독자 우대. 선결제분은 **이연수익(deferred revenue)** 으로 — MRR 아닌 부채로 잡고 인도 시점 인식. 매출은 GMV가 아닌 **소담 순수취(세무사 분배 후 ~7~8%)** 기준으로만 인식.

## 6. GTM·성장 레버 (우선순위)

1. **환급 시즌 캠페인(연 최대 모멘텀, 3~5월)**: "환급 예상액 무료 진단 → PRO 전환". 삼쩜삼이 키운 "환급=당연" 인식에 무임승차하되 **"수수료 캡 + 구독자 우대"**로 차별화.
2. **직원→사장 바이럴(CAC 0)**: 알바가 명세서·출퇴근으로 SODAM 인지 → 명세서 하단 **"이 가게는 SODAM으로 급여를 정확히 지급합니다" 워터마크** = 무료 광고 + 이직 시 신규 사장 역추천.
3. **세무사 파트너 채널(B2B2B)**: 세무사 보유 고객풀에 리퍼럴/화이트라벨. 환급 매출 일부 셰어로 인센티브.
4. **전환 트리거(제품 내 자동 펌프)**: ① 주휴수당 발생 경고(금액 표시) ② 명세서 발급 페이월(월급날) ③ 3번째 직원 추가 ④ 세무시즌.
5. **가격 A/B 우선순위**: ① STARTER 전환율·업셀률(₩9,900 적정성) ② 환급 수수료 vs 번들 LTV ③ FREE 직원 한도 1 vs 2.

## 8. 코드·구현 연동 (개발 백로그)

| 항목 | 현재 | 필요 작업 | 연동 지점 |

| 기능 게이팅 | 없음 | 플랜별 feature gate(직원수·명세서 발급·환급) 미들웨어 | `@MasterOnly`+플랜 검사, FE 페이월 |
| 멀티매장 애드온 | MasterStoreRelation 존재 | 매장수 기반 과금 로직 | `Subscription`, 결제 수량 |
| 명세서 워터마크 | PDF 생성 존재(`generatePayrollPdf`) | FREE용 워터마크 미리보기 변형 | `PayrollService` PDF |
| **환급 정산** | **enum 스텁만, 코드 0%** | **정산 모듈 신설**(환급액 계산·수수료·세무사 분배·이연수익) | 신규 도메인, M6 게이트 |
| 연납·이연수익 | 없음 | 연납 결제·deferred revenue 회계 | 결제(인간 승인), `Subscription` |
| 90일 슬립 | 없음 | 비활성 계정 배치(데이터 보관·연산 중단) | `@Scheduled` 배치 |
