## 0. 요약 (TL;DR)

소담은 **직원 1~5명짜리 한국 소상공인을 위한 모바일 운영 비서**다.
**NFC + GPS 출퇴근 → 급여·명세서 자동계산 → 4대보험·세무 준비**까지 단일 앱으로 처리한다.
해자는 "환급"이 아니라 **직원 쓰는 사장님의 실근로데이터 → 정확한 급여·명세서·4대보험 준비 + 분쟁 증거력**이다.
1인 사업가가 **12개월 내 유료 사업장 보수 floor 52개 / 현실적 목표 130개(구독 MRR 약 234만원)**를 목표로 출시한다.

> 💰 가격·매출·수익 모델은 [수익화·수익모델 확정안 v3.1](./수익화-수익모델-확정안.md)이 **본 PRD §5를 대체·확정**한다.

## 1. 문제 정의 (Problem)

### 1.1 사장님의 진짜 고충
1. **알바 출퇴근을 못 믿는다** — 카톡 보고, 사진 인증, 수기 노트. 부정 인증/대리 출근.
2. **주휴수당·야간·연장 시급 계산이 매월 헷갈린다** — 엑셀 함수 만들었다가 매년 노동법 바뀌면 또 헷갈림.
3. **종합소득세·4대보험 신고가 1년에 한 번씩 도장 깨기** — 세무사는 부담, 혼자는 불안.
4. **알바생도 본인 급여 명세를 못 본다** — 사장이 카톡으로 보내주거나 인쇄해야 함.
5. **노무·세무 정보가 너무 흩어져 있다** — 고용노동부, 국세청, 블로그를 헤매야 답이 나옴.

### 1.2 시장 데이터 (가정·추정)
- 한국 소상공인 약 **400만 사업체** (통계청 2024)
- 그 중 **직원 1~5명** 구간은 약 60% (≈ 240만)
- 시프티·알밤 합산 침투율 5% 미만 → **TAM ₩수천억, SAM ₩500억+**

## 2. 솔루션 개요 (Solution)

| 페인 | 소담의 답 |
|---|---|
| 출퇴근 못 믿음 | **NFC 태그 + GPS 반경 이중 검증** (Haversine 공식) |
| 급여 계산 복잡 | **시급·연장·야간·주휴수당 자동 산출** + 명세서 발급 |
| 세무 신고 부담 | **세무 송객(정액 패키지)** — 세무사 파트너 연결·자료 정리(SW+송객만, 대행은 세무사). 환급 정률 모델은 폐기(사장 종소세는 구조적 비환급) |
| 알바생 정보 비대칭 | **직원 앱 화면** — 본인 출퇴근/급여 명세 직접 조회 |
| 정보 흩어짐 | **노무·세무·정책·팁 인포허브** (이미 LaborInfo/TaxInfo/PolicyInfo/TipInfo 도메인 존재) |

## 6. 기술 스택 (Tech Stack) — 현재 코드 기준

### Backend
- Java 17, Spring Boot 3.4.5, Spring Security
- MySQL 8.0, Redis 6
- JWT (Access 1h / Refresh 7d)
- AWS (EC2/ECS + RDS + ElastiCache 단일 리전)
- OpenAPI 3.0 (`ApiList.yaml`, 100+ 엔드포인트 정의 완료)

### Frontend
- React Native 0.81, React 19
- TypeScript, React Navigation v6, TanStack Query v5
- NFC: `react-native-nfc-manager`
- GPS: `react-native-geolocation-service`
- Charts: `react-native-chart-kit`

### 추가 도입 필요 (출시 전)
- **결제**: 토스페이먼츠 정기결제 (B2B 카드/계좌이체)
- **푸시**: FCM (출퇴근 누락/급여 지급 알림)
- **분석**: Amplitude or 자체 이벤트 로깅 → Redash
- **에러 추적**: Sentry (FE+BE)
- **CS**: 채널톡 (Live Chat 위젯)
