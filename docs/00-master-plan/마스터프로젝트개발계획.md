# 1. CEO 관점 — 비전, 시장, GTM

## 1.1 비전 (Vision)
> **"5년 안에 한국 소상공인 10만 사업장이 매일 켜는 운영 비서가 된다."**

## 1.2 미션
- 사장님이 직원 관리·급여·세금에 쓰는 시간을 **월 4시간 → 월 30분**으로 줄인다.
- 알바생이 "내가 일한 만큼 받았는지" 1초 안에 확인하게 한다.
- 세무사·노무사 비용 부담 없이 소상공인이 합법적으로 운영할 수 있게 돕는다.

## 1.3 시장 진입 전략 (Go-To-Market)

### Phase 1 — 시드 100 (M0~M3)
- **채널**: 당근(동네 사장님 게시판), 인스타 릴스, 소상공인연합회 카페
- **메시지**: "월 1.5만 원으로 노무사 비용을 0으로 만드세요"
- **방법**:
  1. 동네 카페·음식점 100곳 직접 방문 (서울·경기, 주 20곳)
  2. 베타 사용 사장님에게 6개월 무료 + 평생 50% 할인
  3. 인플루언서 자영업 유튜버 3명 시드 미팅

### Phase 2 — 그로스 1,000 (M4~M8)
- 추천 코드(레퍼럴): 추천 시 양쪽 1개월 무료
- 소상공인 카드사 제휴(BC·신한·하나) — 카드 결제 시 1개월 무료
- 종소세 시즌(5월) 마케팅 집중 — "지난해 환급 못 받은 분, 소담이 대신"

### Phase 3 — 스케일 (M9+)
- 세무사 협회 B2B 채널
- 상권 분석 SaaS, POS 업체와 제휴 번들

## 1.4 브랜드 톤 & 메시지
- **톤**: 다정한 동네 친구 같은 사장 (반말 X, 존댓말 + 친근 표현)
- **금지어**: "혁신", "AI 기반", "스마트" — 사장님은 이런 단어 안 좋아함
- **선호어**: "사장님", "월말 30분", "환급", "내 시급", "주휴수당"

## 1.5 경쟁 포지셔닝 (1줄 한정)

# 2. CTO 관점 — 아키텍처, 개발, 인프라

## 2.1 기술 원칙 (Engineering Principles)
1. **모바일 우선** — 모든 기능은 폰에서 끝낸다.
2. **단순 > 영리** — 1인이 유지하므로 마이크로서비스·이벤트 소싱 금지.
3. **자동화 가능한 건 코드로** — 백오피스·CS도 코드로 처리.
4. **외부 위임 가능한 건 외부로** — 결제(토스), 푸시(FCM), CS(채널톡), 분석(Amplitude), 에러(Sentry).
5. **데이터는 자산** — 모든 도메인 이벤트는 로깅 테이블에 적재 (이후 분석/AI 활용).

## 2.2 시스템 아키텍처 (As-Is → To-Be)

### As-Is (현재 코드)
```
┌──────────────────────────────────┐
│  React Native 0.81 (Android FE)  │
│  - JWT, React Query, NFC, GPS    │
└──────────────┬───────────────────┘
               │ HTTPS
┌──────────────▼───────────────────┐
│  Spring Boot 3.4.5 (Monolith)    │
│  - JWT, AOP, JPA, OpenAPI        │
└──────┬───────────────────┬───────┘
       │                   │
┌──────▼──────┐   ┌────────▼──────┐
│  MySQL 8    │   │  Redis 6      │
└─────────────┘   └───────────────┘
```

### To-Be (출시 시점)
```
                  ┌── FCM ─── 푸시
                  ├── 토스페이먼츠 ─── 정기결제
   React Native ──┼── Sentry ─── 에러
                  ├── 채널톡 ─── CS
                  └── Amplitude/자체 ─── 분석
        │
        ▼
   Spring Boot (단일 EC2/ECS, t3.medium)
        │
        ├── RDS MySQL (db.t4g.small, Multi-AZ off → on @M6)
        ├── ElastiCache Redis (cache.t4g.micro)
        ├── S3 (영수증·서류 업로드)
        └── CloudWatch + 알람(SNS → 카톡 봇)
```

## 2.3 개발 우선순위 (P0/P1/P2)

### P0 — 출시 차단 (Blocker)
1. **토스페이먼츠 정기결제** (Subscription 엔티티 + 4 API + 웹훅)
2. **FCM 푸시** (출퇴근 누락, 결제 성공/실패)
3. **Sentry FE/BE 연동**
4. **채널톡 위젯**
5. **.env → AWS Secrets Manager 이관**
6. **개인정보 처리방침 · 이용약관 · 마케팅 동의 화면**
7. **Play Store 등록 자료 (스크린샷·설명·심사용 계정)**
8. **`security_report.md` 보안 이슈 해소**

### P1 — 출시 후 30일 이내
1. 멀티 매장 사장님 대시보드 보강
2. 급여대장 PDF/엑셀 출력
3. 휴가/연차 워크플로우
4. 환급형 플랜 베타 (세무사 1곳 연결)

### P2 — 출시 후 90일 이내
1. iOS TestFlight → App Store
2. 웹 대시보드 베타 (RN Web 활용)
3. 광고 매출 (AdMob)
4. 시급 변경 이력 그래프

## 2.4 코드 품질 가드레일
- BE: Spring Boot Test + JPA Slice Test, 핵심 도메인(Payroll/Attendance) 단위 커버리지 ≥ 70%
- FE: Jest + RN Testing Library, 화면 스냅샷 + 핵심 훅 단위 테스트
- CI: GitHub Actions — Lint → Test → Build → 수동 승인 → Deploy
- DB 마이그레이션: Flyway 도입 (현재 미사용, M1에 도입)
- 시크릿: 커밋 금지 (pre-commit hook + gitleaks 도입)

## 2.5 보안 베이스라인
- 주민번호·계좌번호·카드번호는 **절대 직접 저장 안 함**
- 카드 = 토스 빌링키만 저장
- 비밀번호 = BCrypt (이미 적용)
- 모든 PII 필드 컬럼 암호화 (M2)
- 로그에 PII 마스킹 (이메일 일부, 전화 가운데 자리)
- RDS 암호화 활성화, S3 KMS
- 분기 1회 OWASP ZAP 자체 스캔

## 3.3 단위경제 (Unit Economics)

| 지표 | 값 | 근거 |
|---|---|---|
| CAC (Customer Acquisition Cost) | ₩30,000 | 광고비 / 신규 유료 사장님 |
| ARPU (비즈니스 플랜) | ₩15,000 | 정가 |
| Gross Margin | 93% | (15K - 1K PG) / 15K |
| Avg Customer Lifetime | 18개월 | 카페·식당 평균 폐업률 보정 |
| **LTV** | **₩270K** | ARPU × 라이프타임 × 마진 |
| **LTV / CAC** | **9.0** | ≥ 3 이면 건강, ≥ 6 이면 우수 |
| Payback Period | 2.2개월 | CAC / (ARPU × 마진) |
