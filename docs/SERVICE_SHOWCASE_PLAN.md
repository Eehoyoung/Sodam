# 소담 서비스 소개 캡처 및 HTML 제작 계획

## 목적

실행 중인 소담 Android 앱의 사장(Master)과 직원(Employee) 흐름을 실제 화면으로 캡처하고, 캡처를 근거로 핵심 기능을 설명하는 정적 HTML 소개 페이지를 만든다. 기능의 존재를 설명하는 데 그치지 않고 가입부터 매장 운영과 급여 정산까지 이어지는 사용 흐름을 보여 준다.

## 산출물

- `artifacts/service-showcase/screenshots/`: 역할과 기능별 원본 PNG 캡처
- `artifacts/service-showcase/manifest.md`: 각 이미지의 파일명, 역할, 화면명, 기능 설명, 검증 상태
- `artifacts/service-showcase/index.html`: 캡처를 사용하는 서비스 소개 페이지
- `artifacts/service-showcase/assets/`: 소개 페이지 전용의 비밀정보 없는 정적 자산

캡처와 HTML에는 실사용자 PII, 실제 GPS 좌표, 토큰, 비밀번호, API 키를 포함하지 않는다. 데모 계정과 시드 데이터만 사용한다.

## 실행 및 검증 순서

1. 현재 변경 사항을 보존한 채 Docker Compose, 의존성 설치 상태, Android SDK/AVD, adb 연결 상태를 확인한다.
2. MySQL, Redis, 백엔드를 Docker Compose로 기동하고 API 헬스와 컨테이너 상태를 확인한다.
3. Metro를 포트 8088에서 실행하고 사장용·직원용 Android 에뮬레이터 두 대에 앱을 설치한다. 두 에뮬레이터 모두 `10.0.2.2:8088` Metro 연결을 확인한다.
4. 사장 계정 가입, 매장 등록, 초대 코드 발급을 수행하고 직원 계정 가입과 초대 코드 입사를 연결한다.
5. 각 화면은 PNG와 UI XML을 함께 저장하고, 화면 제목·주요 CTA·목록/상태 값이 실제로 표시되는지 확인한다.
6. 캡처 파일을 역할별 사용자 여정과 기능 영역으로 분류해 manifest에 기록한다.
7. 모바일과 데스크톱에서 읽기 좋은 정적 HTML을 구현한다. 모든 소개 항목은 실제 캡처 이미지와 한 줄 기능 설명을 연결한다.
8. HTML을 브라우저에서 렌더링하여 이미지 누락, 텍스트 넘침, 모바일 레이아웃을 검증한다.

## 필수 캡처 목록

### 공통 및 가입

- 시작/로그인 화면
- 사장 역할 선택 및 회원가입
- 직원 역할 선택 및 회원가입
- 위치 정보 수집 동의 화면
- 사장 매장 등록
- 직원 초대 코드 입사

### 사장 핵심 운영

- 사장 대시보드 및 실시간 출퇴근 현황
- 매장 상세와 출퇴근 인증 방식/GPS 반경 설정
- 직원 목록 및 직원 상세(임금 정보는 데모 값만)
- 초대 코드 발급 또는 재발급
- 주간 드래그앤드롭 스케줄 보드
- 스케줄 확정
- 출퇴근 승인 요청 처리
- 휴가/결근 요청 승인
- 근로계약서 작성·전송 또는 전자서명 현황
- 급여 정산 3단계 중 계산 검토와 확정
- 급여 명세서 또는 급여 이력
- 즉시 보너스
- 근로 리스크 경고 또는 매장 인사이트
- 매장 공지

### 직원 핵심 이용

- 직원 홈과 오늘의 근무/스케줄
- GPS 또는 사장 승인 방식의 출근
- 퇴근 및 휴게시간 기록
- 내 주간 스케줄
- 교대 요청
- 휴가/결근 신청
- 전송된 근로계약서 확인 및 서명
- 내 급여/명세서 이력
- 공지 및 알림
- 내 정보/매장 소속 화면

## 캡처 기준

- 코어 흐름인 `가입 → 매장 등록/입사 → 출퇴근 → 스케줄 → 급여`는 반드시 양 역할 관점에서 포함한다.
- 가능한 모든 구현 완료 기능을 추가 캡처하되, 미구현 또는 오류 상태를 기능 완료처럼 소개하지 않는다.
- 캡처 파일명은 `NN-role-feature-state.png` 형식을 사용한다. 예: `08-master-schedule-board.png`.
- 각 기능 전환 후 UI XML을 함께 보관하여 화면의 실제 표시 상태를 확인한다.
- 화면에 개인 정보가 표시될 수 있으면 데모 데이터로 교체하거나 해당 영역을 제외하고 캡처한다.

## HTML 구성

1. 서비스명과 대상: 소상공인 매장의 출퇴근·일정·급여 관리
2. 사장과 직원이 연결되는 핵심 여정
3. 사장 기능 갤러리: 매장·직원·출퇴근·스케줄·계약·급여·공지/인사이트
4. 직원 기능 갤러리: 입사·출퇴근·휴게·일정·요청·계약·급여·알림
5. 운영 흐름: 출퇴근 기록에서 급여 확정 및 명세서까지
6. 실제 앱 화면 모음

소개 문구는 캡처에서 확인된 동작만 기술한다. HTML은 외부 비밀값이나 외부 CDN 의존 없이 로컬 이미지로 열 수 있도록 만든다.

## 완료 조건

- Docker, Metro, 사장·직원 에뮬레이터 두 대가 동시에 정상 동작한다.
- 사장·직원 회원가입부터 직원 입사까지 연결된 시나리오가 재현된다.
- 코어 기능 5종(매장/입사, 출퇴근, 스케줄, 급여, 계약 또는 승인)이 모두 실제 캡처로 소개된다.
- 가능한 추가 기능을 포함해 역할별 최소 10장, 전체 30장 이상의 유효 캡처를 확보한다. 개발된 기능 목록에 표시된 각 기능은 서로 다른 상태 또는 역할 관점의 실제 화면을 2장 이상 확보한다.
- `index.html`에서 모든 캡처가 렌더링되고 데스크톱·모바일에서 텍스트와 이미지가 겹치지 않는다.

## 진행 현황 (2026-07-18)

- Docker Compose(백엔드·MySQL·Redis), Metro 8088, Android 에뮬레이터 2대가 동시에 실행 중이다.
- 사장 계정, 직원 계정, 매장 소속을 연결하고 직원 위치 출근을 실행했다. 출근 등록은 백엔드와 사장 대시보드·알림함에 반영됐다.
- PNG는 Android 내부 `screencap` 후 `adb pull` 방식으로 재수집 중이다. PowerShell 표준 출력 리디렉션으로 생성된 초기 PNG는 바이트 변환 문제가 확인되어 소개 HTML에서 제외했다.
- 현재 PNG 시그니처를 검증한 캡처는 36장이고, 소개 HTML은 그중 23개 실제 화면을 참조한다. 전체 30장 목표는 충족했으며, 기능별 2장 커버리지를 계속 보강한다.
- 사장 수동 기록 진입의 직원 전용 API 404, 위치 동의 전 출근 요청의 개발 콘솔 오류 노출, 직원 화면의 출근 당일 상태 갱신 문제는 `artifacts/service-showcase/manifest.md`의 결함 증빙으로 분리했다.
